# Experiments/ — 实战项目目录

> 4 个项目作为简历亮点：企业 RAG / Multi-Agent / Function Calling / Agent 评估。

## 项目结构

每个项目一个子目录，包含：

```
Experiments/<project-id>-<name>/
├── README.md          # 项目说明（目标 / 技术栈 / 演示）
├── design.md          # 架构设计（系统图 / 数据流 / 关键决策）
├── docs/              # 设计文档 / API 文档
├── src/               # 源代码
├── tests/             # 测试
├── demo/              # 演示视频 / 截图
└── requirements.txt   # 依赖
```

## 完成标准

每个项目必须满足：

- ✅ README.md（含演示 GIF / 视频链接）
- ✅ design.md（含系统架构图）
- ✅ 完整可运行代码（`python -m <project>` 能跑）
- ✅ 至少 5 个核心功能 demo
- ✅ 单元测试覆盖率 ≥ 60%
- ✅ GitHub 仓库（README 含项目介绍 + 技术栈 + 启动命令 + 演示）
