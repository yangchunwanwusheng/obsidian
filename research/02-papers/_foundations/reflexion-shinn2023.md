# Reflexion: Language Agents with Verbal Reinforcement Learning

> 一种基于语言反馈的强化学习方法，通过让语言 Agent 进行自我反思来持续改进性能，无需模型微调即可实现迭代提升

## 论文概述

**Reflexion（反思）** 是由东北大学（Northeastern University）和普林斯顿大学（Princeton University）的研究者于 2023 年发表的研究工作，发表于 NeurIPS 2023（Conference on Neural Information Processing Systems，神经信息处理系统大会）。这篇论文提出了一个新颖的框架，让语言 Agent（Language Agent）能够通过**语言强化学习（Verbal Reinforcement Learning）** 机制持续改进自身性能，而无需对底层模型进行权重更新。

传统的强化学习（Reinforcement Learning, RL）方法，如 Policy Gradient（策略梯度）或 Q-Learning，需要通过数值奖励（numerical reward）信号来更新模型的参数（weights）。这种方法在游戏、机器人控制等场景中效果显著，但存在几个关键问题：

首先，**需要大量样本**：RL 通常需要数以万计的交互样本才能学会一个任务，这对于需要复杂推理的语言 Agent 来说成本过高。

其次，**奖励函数设计困难**：对于许多任务，尤其是涉及语言输出的任务，设计一个能够准确评估输出质量的奖励函数（reward function）非常困难。

第三，**无法利用语言中的丰富信息**：数值奖励只能传达"好"或"坏"的信息，但无法说明"为什么"或"如何改进"。

Reflexion 的核心创新在于：它用**语言反馈（verbal feedback）** 替代了传统的数值奖励，用**记忆系统（memory）** 替代了模型权重的更新。具体来说，当 Agent 产生一个错误时，不是通过梯度下降来调整参数，而是让 Agent 用自然语言反思错误的原因，并将这个反思存入记忆；在下一次尝试时，Agent 会读取之前的反思，避免重复同样的错误。

这种方法的优势是显著的：不需要大量样本（因为没有参数更新），不需要设计复杂的奖励函数（因为反馈是语言描述的），并且能够获得详细的改进指导（因为反思说明了问题所在）。

## 核心架构：三大组件

Reflexion 的架构由三个核心组件构成：**Actor（行动者）**、**Evaluator（评估者）** 和 **Self-Reflection（自我反思）**。这三个组件形成了一个闭环的迭代改进系统。

### 1. Actor（行动者）

**Actor（行动者）** 是 Reflexion 框架中负责生成动作（action）的组件。在语言 Agent 的语境下，动作可以是：

- 对用户问题的回答
- 执行一个工具调用（如搜索网页、计算数学表达式）
- 在环境中采取一个物理动作（如在文本游戏中移动）
- 生成一段代码

Actor 可以基于多种推理模式工作：

**Chain-of-Thought（CoT，思维链）**：Actor 在生成最终动作之前，先显式地一步步推理，展示推理过程。这种方法最早由 Google 提出，能够显著提高复杂推理任务的效果。

**ReAct**：结合了 Reasoning（推理）和 Acting（行动）。Actor 不仅进行推理，还会在推理过程中调用外部工具或环境交互，将推理结果用于指导下一步行动。

Reflexion 的设计是通用的，不依赖于特定的推理模式。论文中展示了使用 CoT 和 ReAct 都能从 Reflexion 框架中受益。

Actor 的输出不仅包括最终的答案或动作，还包括一个**轨迹（trajectory）**——从初始状态到最终状态的完整推理和行动序列。这个轨迹会被送往 Evaluator 进行评估。

### 2. Evaluator（评估者）

**Evaluator（评估者）** 负责评估 Actor 产生的轨迹，并给出奖励信号（reward signal）。这个组件的存在使得 Reflexion 能够在不需要人工标注的情况下进行自我改进。

Evaluator 有多种实现方式：

**Ground Truth 评估**：当任务有明确的正确答案时（如数学题、代码执行结果），Evaluator 可以直接将 Agent 的输出与正确答案比较，给出二元（binary）的对/错奖励。

**Self-hosted Eval**：对于没有标准答案的任务（如"写一首诗"），Evaluator 本身也是一个 LLM，它会根据预设的标准（如"是否押韵"、"情感是否连贯"）对输出进行评分。

**Human Proxy Eval**：模拟人类评估者的判断，使用 LLM 来模拟人类对输出质量的主观判断。

Evaluator 的输出不仅是一个数值分数，还可能包括对轨迹的详细分析——哪些步骤是正确的，哪些是错误的，哪里有改进空间。这些分析会作为输入传递给 Self-Reflection 组件。

### 3. Self-Reflection（自我反思）

**Self-Reflection（自我反思）** 是 Reflexion 的核心创新组件，也是区别于其他 Agent 框架的关键所在。当 Evaluator 判定 Actor 的当前尝试不够好时，Self-Reflection 会生成一段语言反思（verbal reflection），分析错误的原因，并给出改进建议。

Self-Reflection 的输入包括：
- 当前任务的描述
- Actor 生成的完整轨迹
- Evaluator 给出的反馈

Self-Reflection 的输出是一段自然语言反思，内容可能包括：
- "在第三步，你错误地假设了 X，实际上应该是 Y"
- "你应该先检查前提条件，而不是直接应用公式"
- "这个问题类似于之前遇到的 Z，但关键区别在于..."

这些反思内容会被存储到一个**记忆缓冲区（memory buffer）** 中。在 Reflexion 的设计中，这个记忆缓冲区是**循环的（cyclic）**——它有固定的大小，当新的反思写入时，最旧的反思会被覆盖。这种设计确保了 Agent 的反思始终是相关的和最新的。

在下一次任务迭代中，Actor 在生成轨迹之前，会先读取记忆缓冲区中的反思内容，将其作为额外的上下文。这使得 Actor 能够：
- 避免重复之前犯过的错误
- 参考之前的成功经验
- 保持对任务目标的清晰认知

## 循环记忆缓冲区（Cyclic Memory Buffer）

**Cyclic Memory Buffer（循环记忆缓冲区）** 是 Reflexion 中存储反思结果的数据结构。它的设计灵感来自于人类认知中的工作记忆（working memory）——人类能够保留的信息量是有限的，旧的信息会逐渐被新的信息替代，除非被强化为长期记忆。

在 Reflexion 中，这个缓冲区有预定义的大小（比如 1-10 条反思）。每次 Agent 在某个任务上失败并产生新的反思时，这条反思会被添加到缓冲区中。如果缓冲区已满，新的反思会覆盖最旧的反思，形成一个循环。

这种设计的考虑是：

**避免记忆过载**：如果 Reflexion 试图记住所有过去的反思，记忆会变得臃肿且不相关。最新的反思通常是最有价值的。

**促进策略更新**：当 Agent 在某个策略上持续失败时，旧的、错误的策略假设会被新的反思覆盖，推动 Agent 尝试新的方法。

**模拟人类遗忘**：人类不会记住每一个细节，尤其是那些不重要的或已经被替代的信息。循环缓冲区模拟了这种自然的遗忘机制。

当然，这种设计也有潜在的代价——一些可能有价值的长期经验可能会被意外覆盖。论文中没有详细讨论这个问题，但未来工作可以探索将重要的反思"晋升"为长期记忆的机制。

## 语言强化学习（Verbal Reinforcement Learning）

Reflexion 提出的**Verbal Reinforcement Learning（语言强化学习）** 是一个介于传统 RL 和纯推理之间的范式。它共享了 RL 的核心思想——通过试错和奖励信号来改进策略——但用语言反馈替代了数值奖励，用记忆更新替代了权重更新。

### 与传统 RL 的对比

| 方面 | 传统 RL | Reflexion (VRL) |
|-----|---------|----------------|
| 奖励信号 | 数值（标量） | 语言（文本） |
| 更新方式 | 梯度下降修改权重 | 修改提示词/记忆 |
| 样本效率 | 通常需要大量样本 | 样本高效，通常几次迭代即可 |
| 可解释性 | 难以解释为什么有效 | 反思文本清楚说明了改进方向 |
| 适用场景 | 简单动作空间 | 复杂语言任务 |

### 语言反馈的优势

使用语言作为反馈媒介有以下几个关键优势：

**信息丰富**：语言可以传达细致的、情境化的反馈。"你的答案是错的，因为你在计算中没有考虑负数的情况" 比单纯返回一个 -1 的奖励信号包含更多信息。

**无需训练**：传统 RL 需要大量的交互样本来估计值函数（value function）或更新策略网络。VRL 不需要任何参数更新，因此不存在过拟合或训练不稳定的问题。

**快速迭代**：每次试错后，Agent 可以立即根据反馈调整策略，而不需要等待模型收敛。

**可解释性强**：Agent 的反思过程是显式的，可以被人类理解和检查。这对于构建可信赖的 AI 系统很重要。

### 收敛性讨论

一个自然的问题是：如果 Reflexion 只是修改提示词而不更新模型权重，它能收敛到正确的解吗？论文通过实验表明，在许多任务上答案是"能"。原因在于：

当 Agent 在提示词中看到"不要用方法 X，因为它在 Y 情况下会失败" 这样的反思时，它会倾向于避免这种错误。随着迭代次数的增加，Agent 通过记忆积累了对任务的越来越准确的理解，最终能够做出正确的决策。

这本质上类似于人类的学习过程：我们不是通过修改大脑神经元的权重来学习骑自行车，而是通过回忆"上次为什么摔倒"并在下一次尝试时调整行为。

## 实验结果

Reflexion 论文在多个基准任务上进行了实验，展示了该方法的有效性。

### 1. HumanEval（代码生成）

**HumanEval** 是 OpenAI 提出的代码生成评估基准，包含 164 个 Python 编程题。模型需要根据文档字符串（docstring）生成能够通过单元测试的代码。

Reflexion 在这个任务上取得了 **91% pass@1** 的惊人成绩。所谓 pass@1 是指模型第一次生成（采样一次）的代码就能通过测试的比例。

对比基线：
- GPT-4 无 Reflexion：约 67%
- Codex 无 Reflexion：约 47%
- 加入 Reflexion 后：显著提升至 91%

这个结果说明了 Reflexion 能够有效地帮助 Agent 从之前的代码错误中学习，逐步改进生成的代码质量。

### 2. AlfWorld（文本决策）

**AlfWorld** 是一个基于文本的决策任务环境，包含各种需要多步骤推理和规划的居家任务（如"找到并拿起刀"、"把苹果放到厨房的柜子里"等）。

Reflexion 在 AlfWorld 上达到了 **97% 成功率**，几乎完美。这个结果特别有意义，因为 AlfWorld 任务需要 Agent 进行序列决策（sequential decision-making），每一步的选择都会影响后续的可用选项。

极高的成功率表明 Reflexion 的反思机制能够帮助 Agent：
- 追踪已完成和未完成的子目标
- 识别之前尝试失败的原因
- 调整策略直到成功

### 3. 持续改进分析

论文还特别分析了 Reflexion 在多轮迭代中的表现。结果显示：

- **前几次迭代改进显著**：从第一次尝试到第三次尝试，性能提升最为明显
- **持续改进**：即使经过多次迭代，Reflexion 仍能带来额外的提升，表明反思机制不是在第一次就饱和的
- **错误类型收敛**：随着迭代增加，某些类型的错误完全消失，表明反思有效地帮助 Agent 建立了正确的认知

## 技术实现细节

### 完整流程

一次 Reflexion 迭代的完整流程如下：

1. **任务接收**：Actor 接收任务描述（如"解决这个数学问题"）
2. **记忆读取**：Actor 从记忆缓冲区读取之前的反思
3. **轨迹生成**：Actor 结合任务描述和历史反思，生成一个完整的解决方案轨迹
4. **评估**：Evaluator 对轨迹进行评估，判断是否成功
5. **反思生成**：如果失败，Self-Reflection 根据反馈生成新的反思
6. **记忆更新**：新的反思被写入记忆缓冲区，覆盖旧内容
7. **下一轮**：返回步骤 2，继续迭代

这个流程会重复直到任务成功或达到最大迭代次数。

### 与 ReAct 的结合

Reflexion 可以与多种推理框架结合，论文特别展示了与 **ReAct** 的结合。

ReAct 的核心思想是让 LLM 在推理过程中交替进行"思考"（thought）和"行动"（action）。"思考"是对当前状态的分析，"行动"是对环境的具体操作（如调用工具）。

当 Reflexion 与 ReAct 结合时：
- Actor 在生成"思考"时会参考记忆缓冲区中的反思
- Evaluator 评估完整的 Thought-Action-Observation 轨迹
- Self-Reflection 分析推理链中的具体错误

这种组合在 AlfWorld 等需要工具使用和多步骤推理的任务中特别有效。

### 实现复杂度

Reflexion 的实现相对简单，不需要复杂的深度学习框架或强化学习库。核心组件是：
- 一个 LLM API（如 OpenAI 的 GPT-4）
- 一个记忆数据结构（简单的列表）
- 几个提示词模板

这种简洁性使得 Reflexion 易于复现和扩展。

## 论文的局限性与未来方向

### 局限性

**长期记忆缺失**：循环记忆缓冲区的大小有限，对于需要在极长时间跨度内积累知识的任务，当前设计可能不够。

**自我欺骗风险**：如果 LLM 的反思包含错误或不准确的分析，Agent 可能会形成错误的心智模型。缺乏外部纠正机制可能导致"一错再错"。

**评估器依赖**：Reflexion 的效果很大程度上依赖于 Evaluator 的质量。如果 Evaluator 给出错误的反馈，反思可能误导 Agent。

**不适用于所有任务**：对于某些完全随机的任务（如抛硬币），反思无法带来任何改进，因为不存在可学习的模式。

### 未来方向

论文提出的未来研究方向包括：

- **多 Agent 反思**：多个 Agent 相互提供反馈，形成更丰富的反思视角
- **记忆层次结构**：引入 Generative Agents 中的层次化记忆，将短期反思与长期洞察分离
- **外部知识集成**：将 Reflexion 与外部知识库结合，使 Agent 能够验证和纠正自己的反思
- **记忆固化**：探索将反复验证为正确的反思"固化"为系统提示词的一部分

## 对 Agent Memory 领域的意义

Reflexion 为 Agent Memory 研究贡献了几个重要的思想：

1. **语言作为记忆格式**：不是所有记忆都需要是离散的键值对，语言反思可以是一种有效的记忆形式。

2. **遗忘的价值**：循环缓冲区提醒我们，适度的遗忘可能是智能系统的必要特性，而非缺陷。

3. **无需训练的改进**：证明了可以通过改进提示词和记忆来提升性能，而不需要每次都微调模型。

4. **自我改进能力**：展示了 AI 系统可以通过积累经验实现自我改进的潜力。

## 核心要点回顾

1. **Reflexion** 框架包含三个核心组件：Actor（行动者）、Evaluator（评估者）和 Self-Reflection（自我反思）

2. **Cyclic Memory Buffer（循环记忆缓冲区）** 以固定大小的循环列表存储语言反思，新的反思覆盖旧的

3. **Verbal Reinforcement Learning（语言强化学习）** 用语言反馈替代数值奖励，用记忆更新替代权重更新

4. Reflexion 在 HumanEval 达到 **91% pass@1**，在 AlfWorld 达到 **97% 成功率**

5. 优势：样本高效、无需训练、可解释性强

6. 该工作展示了通过语言反馈实现 Agent 自我改进的可能性
