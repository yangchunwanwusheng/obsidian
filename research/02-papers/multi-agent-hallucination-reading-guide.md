# 多智能体幻觉：文献导读与推荐阅读路线

> 本文档整理了"多智能体系统中 LLM 幻觉问题"方向的核心文献，按由浅入深的阅读路线组织，供系统性入门与深耕使用。所有 PDF 存放于 `raw/assets/papers/`。

---

## 一句话理解这个方向

当多个 LLM 智能体协作时，单个智能体的幻觉会**传播、放大、级联**到整个系统——这就是"多智能体幻觉"。核心挑战是：**如何在保持协作效率的同时，检测并抑制幻觉的传播？**

---

## 推荐阅读路线

### 🟢 Phase 1：建立问题认知（2-3 天）

> 目标：理解"多智能体幻觉"是什么、为什么重要、现有方法有哪些局限

#### 1. MetaGPT — 问题定义的起点

- **文件**：`MetaGPT_2023.pdf`
- **为什么先读**：引用 1700+，是该领域最早系统提出"级联幻觉"问题的论文。它指出：多个智能体简单串联时，幻觉会像多米诺骨牌一样逐级放大
- **核心观点**：将人类 SOP（标准操作流程）编码进 prompt，用模块化输出验证来抑制幻觉传播
- **阅读重点**：§1 Introduction + §3.2 Cascading Hallucination + §4 Framework
- **读完你应该能回答**：为什么多智能体比单智能体更容易产生严重幻觉？

#### 2. Stop Overvaluing MAD — 批判性思考

- **引用**：21（Semantic Scholar 可查）
- **为什么接着读**：在你了解 MAD 的美好愿景之后，这篇论文会泼一盆冷水——它系统证明了 MAD **经常不如**单智能体的 Chain-of-Thought
- **核心发现**：5 种 MAD 方法 × 9 个基准 × 4 个基础模型，大部分场景下 MAD 输不掉 CoT/SC
- **关键洞见**：**模型异质性**（heterogeneity）是让 MAD 真正有效的关键因素
- **阅读重点**：§3 Experimental Setup + §4 Results + §5 Heterogeneity Analysis
- **读完你应该能回答**：什么条件下 MAD 才值得用？模型异质性为什么重要？

### 🟡 Phase 2：理解核心技术手段（4-5 天）

> 目标：掌握当前解决多智能体幻觉的三大技术路线——检测、辩论优化、共识机制

#### 3. GUARDIAN — 基于时序图的幻觉检测

- **文件**：`GUARDIAN_2025.pdf`
- **核心思路**：把多智能体协作过程建模为**离散时序属性图**，用无监督编码器检测异常节点（产生幻觉的智能体）
- **亮点**：不用标注数据，利用信息瓶颈理论压缩交互图，捕捉幻觉传播的动态模式
- **阅读重点**：§3 Temporal Graph Modeling + §4 Anomaly Detection
- **读完你应该能回答**：如何用图结构表示多智能体交互？异常检测如何定位幻觉源头？

#### 4. CortexDebate — 稀疏辩论图（ACL 2025）

- **文件**：`CortexDebate_2025.pdf`
- **核心思路**：受人脑白质启发，不是所有智能体都互相辩论，而是构建**稀疏辩论图**——每个智能体只和"对自己有帮助的"智能体交流
- **解决的问题**：① 输入上下文过长导致智能体"迷失"；② 过度自信的智能体主导辩论
- **阅读重点**：§3 Sparse Debating Graph + §4 McKinsey Trust Formula
- **读完你应该能回答**：稀疏通信为什么反而比全连接更有效？

#### 5. MARCH — 多智能体强化学习自检（2026 最新）

- **文件**：`MARCH_2026.pdf`
- **核心思路**：三智能体流水线（Solver → Proposer → Checker），关键创新是**信息不对称**——Checker 只看到分解后的原子命题和证据，看不到 Solver 原始输出，打破自确认偏误
- **亮点**：用 MARL 让三个智能体共同进化，8B 参数模型达到闭源模型水平
- **阅读重点**：§3 Framework + §3.3 Information Asymmetry + §4 Experiments
- **读完你应该能回答**：信息不对称如何解决"自己验证自己"的偏误？

#### 6. G2CP — 图接地通信协议（2026 最新）

- **文件**：`G2CP_GraphGroundedCommunication_2026.pdf`
- **核心思路**：重新设计智能体之间的"语言"——不再用自然语言自由通信，而是用**图操作**（添加节点、添加边、更新属性）作为结构化通信协议
- **解决的问题**：① 自然语言通信中的级联幻觉（信息在传递中被扭曲）；② 自由文本通信的 token 开销巨大
- **亮点**：消除级联幻觉、通信 token 减少 73%、任务完成准确率提升 34%
- **阅读重点**：§3 Graph-Grounded Protocol + §4 Cascading Hallucination Elimination + §5 Experiments
- **读完你应该能回答**：为什么结构化通信能消除级联幻觉？图操作比自然语言通信好在哪里？

#### 7. DebUnc — 不确定性感知辩论（2024）

- **文件**：`DebUnc_HallucinationUncertainty_2024.pdf`
- **核心思路**：在多智能体辩论中，让每个智能体不仅传递答案，还要传递自己的**不确定性估计**（confidence/uncertainty），并用不确定性感知的注意力权重聚合多智能体意见
- **解决的问题**：传统 MAD 中所有智能体"平等发言"，但确定性高的正确答案可能被不确定性高的幻觉"稀释"
- **亮点**：不确定性高的智能体权重降低，有效减少幻觉传播
- **阅读重点**：§3 Uncertainty Estimation in Prompts + §4 Attention-based Aggregation
- **读完你应该能回答**：如何在辩论中量化智能体的"靠谱程度"？不确定性加权为什么比平等投票更有效？

#### 8. MUG — 反事实测试识别幻觉智能体

- **文件**：`MUG_MultiAgentUndercoverGaming_2025.pdf`
- **核心思路**：把 MAD 重构为"谁是卧底"游戏——通过修改输入图像引入**反事实证据**，观察哪些智能体无法识别修改，从而定位"幻觉智能体"
- **三大创新**：反事实验证 > 统计共识；动态修改证据 > 静态输入；主动探测 > 被动回答
- **阅读重点**：§3 MUG Protocol + §4 Counterfactual Test
- **读完你应该能回答**：如何用反事实测试区分"正确推理"和"碰巧猜对"？

### 🔵 Phase 3：深入优化细节（3-4 天）

> 目标：了解通信拓扑、模型多样性、问题漂移等关键优化方向

#### 9. Council Mode — 异构共识框架

- **文件**：`CouncilMode_2026.pdf`
- **核心思路**：将查询分发给多个**异构前沿 LLM** 并行生成，再由专门的共识模型合成输出
- **结果**：幻觉率降低 35.9%，TruthfulQA 提升 7.8 分，代价是 4.2x token 开销
- **阅读重点**：§3 Pipeline（Triage → Generation → Synthesis）+ §5 Cost Analysis

#### 10. Sparse Communication Topology — 通信拓扑研究

- **文件**：`SparseCommMAD_2024.pdf`
- **核心思路**：系统研究了全连接、环形、星形等拓扑对 MAD 的影响，发现**稀疏拓扑可匹配或超越全连接**
- **阅读重点**：§3 Communication Topology + §4 Experiments

#### 11. Problem Drift — 问题漂移现象

- **文件**：`ProblemDriftMAD_2025.pdf`
- **核心思路**：首次定义"**问题漂移**"——辩论越久越偏离原始问题。生成任务漂移率 76-89%，高复杂度任务 7-21%
- **阅读重点**：§3 Problem Drift Definition + §4 Human Expert Analysis + §5 DRIFTPolicy

#### 12. Diversity of Thought — 模型多样性的力量

- **文件**：`DiversityOfThoughtMAD_2024.pdf`
- **核心发现**：中等能力异构模型组合（Gemini-Pro + Mixtral + PaLM2）辩论后 91% 准确率，**超越 GPT-4 单独的 82%**
- **阅读重点**：§3 Experimental Setup + §4 Results

### 🟣 Phase 4：扩展到多模态与专用场景（2 天）

> 目标：了解多智能体幻觉在视觉语言模型等跨域场景中的表现

#### 13. InEx — 内省 + 跨模态协作

- **文件**：`InEx_HallucinationMitigation_2025.pdf`
- **核心思路**：基于熵的不确定性估计触发内省推理，然后跨模态多智能体（编辑 + 自反思）迭代验证，无需训练

#### 14. Interpreting Hallucination in MLLMs via MAD

- **文件**：`InterpretingMitigatingHallucinationMAD_2024.pdf`
- **核心思路**：多模态幻觉 = 缺乏"慢思考"和"发散思考"，用自反思促进慢思考、MAD 促进发散思考，同时**解释**幻觉成因

## 深耕建议

1. **最容易出成果的切入点**：当前文献共识是"模型异质性是关键"，但缺乏理论解释——**为什么异构优于同构？** 这里有形式化空间
2. **蓝海方向**：G2CP 开辟了"结构化通信协议"新范式——用图操作替代自然语言通信，彻底消除级联幻觉。这一方向可进一步探索：不同的结构化表示（知识图谱、逻辑表达式、程序代码）对幻觉抑制的效果差异
3. **理论交叉方向**：GUARDIAN 的时序图方法 + G2CP 的图通信协议 → 可以用图论/网络科学统一"幻觉检测"和"幻觉预防"两个子方向
4. **不确定性方向**：DebUnc 表明将不确定性显式建模进多智能体通信可减少幻觉——这与贝叶斯深度学习、认知不确定性/偶然不确定性分解等经典理论有自然连接
5. **工程价值**：Problem Drift 和 Sparse Topology 的发现对实际系统设计有直接指导意义；G2CP 的 73% token 节省对大规模部署有直接经济价值
6. **建议精读顺序**：MetaGPT → G2CP → MARCH → CortexDebate → DebUnc → GUARDIAN → Stop Overvaluing MAD
