## 二、多智能体协作方案

### 2.1 用户常见提示词场景与对应的协作流程

本方案针对三种核心场景设计：

#### 场景 A：「有规划地续写 N 章内容」（最常用）

**用户输入示例**：「续写3章，从第30章开始」

**协作流程**：

| 步骤 | 执行者 | 动作 | 产出 |
|------|--------|------|------|
| 1. 信息收集 | **主智能体** | 读取：①当前卷纲（04）②最近5章正文 ③伏笔追踪表（08）④角色当前状态 | 上下文摘要 |
| 2. 剧情规划 | **planner** | 基于卷纲和伏笔状态，规划N章的核心事件、爽点/刀子位置、断章设计 | 逐章大纲（每章200字） |
| 3. 章纲确认 | **主智能体** | 将逐章大纲展示给用户，确认或调整 | 定稿章纲 |
| 4. 正文写作 | **主智能体** | 按章纲逐章写作，每章 3500-4500 字 | 正文 .md 文件 |
| 5. 质量审查 | **code-reviewer** | 检查：①角色行为一致性 ②伏笔回温/埋设是否到位 ③断章效果 ④文风统一度 | 审查报告 |
| 6. 文档更新 | **主智能体** | 更新伏笔追踪表（新增伏笔、状态变更、金句记录） | 更新后的 08/11 文档 |

**关键原则**：
- planner 只在「规划」步骤参与，不参与实际写作
- 写作时必须参考伏笔追踪表，确认哪些伏笔需要在这几章回温
- 每章写完后立即更新追踪表，不要攒着

#### 场景 B：「设计新一卷的卷纲」

**用户输入示例**：「设计卷二的详细卷纲」

**协作流程**：

| 步骤 | 执行者 | 动作 | 产出 |
|------|--------|------|------|
| 1. 现状分析 | **主智能体** | 读取全部纲要文档，梳理卷一遗留的伏笔和角色状态 | 现状报告 |
| 2. 架构规划 | **planner + architect** | 设计卷二的核心冲突、角色弧光、伏笔埋设/回收计划 | 卷纲草案 |
| 3. 技法校准 | **主智能体** | 对照叙事技法规划（05）和写作教程，校准燃刀节奏 | 技法建议 |
| 4. 用户确认 | **主智能体** | 展示给用户，收集反馈 | 定稿卷纲 |
| 5. 文档更新 | **主智能体** | 写入 04-卷纲设计，更新 08-伏笔追踪表 | 更新文档 |

#### 场景 C：「修改/优化已有章节」

**用户输入示例**：「优化第15章的结尾」或「第22章太水了，改一下」

**协作流程**：

| 步骤 | 执行者 | 动作 |
|------|--------|------|
| 1. 定位问题 | **主智能体** | 读取目标章节，对比章纲，识别偏差 |
| 2. 制定修改方案 | **主智能体** | 直接处理（单章修改不需要 planner） |
| 3. 修改执行 | **主智能体** | 改写目标部分 |
| 4. 连带检查 | **主智能体** | 确认修改不影响前后章节的衔接 |

### 2.2 子智能体分工总览

| 子智能体              | 触发条件          | 职责                  | 模型           |
| ----------------- | ------------- | ------------------- | ------------ |
| **planner**       | 续写N章、设计卷纲、卡文时 | 剧情规划、节奏设计、伏笔管理      | GLM-5.1      |
| **architect**     | 设计卷纲、重大设定变更   | 架构层面决策、多线交织设计       | GLM-5.1      |
| **code-reviewer** | 每次写完章节后       | 质量审查（一致性、文风、伏笔）     | MiniMax-M2.7 |
| **主智能体**          | 所有步骤          | 信息收集、正文写作、文档更新、用户交互 | MiniMax-M2.7 |

## 三、上下文管理策略

### 3.1 问题：百万字长篇的上下文爆炸

写完 150-200 万字后，不可能每次写作都把全部正文塞进上下文。需要一套**分层摘要 + 按需加载**的策略。

### 3.2 三层上下文架构

| 层级 | 内容 | 大小 | 加载时机 |
|------|------|------|---------|
| **常驻层** | 纲要文档（00-06）、角色设定（02）、力量体系（01） | 约 3 万字 | 每次写作必带 |
| **活跃层** | 最近 5 章正文 + 当前卷纲 + 伏笔追踪表 | 约 2-3 万字 | 每次写作必带 |
| **归档层** | 更早的章节正文 | 每章 3500-4500 字 | 按需检索 |

**常驻层的内容建议定期压缩**：
- 纲要文档不需要全文加载，可以提炼出「精简版」供日常使用
- 例如：世界观设定（01）全文约 7000 字，可压缩为 2000 字的「快速参考卡」

### 3.3 上下文压缩/归档节奏

| 节点 | 动作 | 原因 |
|------|------|------|
| **每 10 章** | 生成一份「10 章摘要」（约 500 字） | 压缩早期正文，保留关键信息 |
| **每卷结束** | 生成「卷摘要」（约 1000 字）+ 更新所有纲要文档 | 跨卷时不再加载上一卷正文 |
| **每 50 章** | 检查伏笔追踪表，归档已回收伏笔 | 减少追踪表体积 |

### 3.4 「10 章摘要」模板

每写完 10 章，由主智能体生成一份摘要，存入 `人间镇陵人/小说纲要内容/` 目录：

```markdown
## 第1-10章摘要

**主线推进**：沈夜觉醒神力，发现护身符是太昊陵碎片，母亲之死与灵异有关。
**角色变化**：沈夜从懦弱→被迫踏入；苏黎体温锚定效应初现；周远开始显露异常。
**关键伏笔**：F001碎片来历、F003周远知情、F008苏黎能力、F009体温锚定。
**代价清单**：第3章短暂忘记周远的脸。
**未解悬念**：H001母亲死因、H003太昊陵深处、H005"你终于来了"。

## 第11-20章摘要
（类似格式）

## 第21-30章摘要
（类似格式）
```

### 3.5 按需检索策略

写作过程中如需引用早期章节的具体内容：

| 需求 | 方法 | 示例 |
|------|------|------|
| 确认某个伏笔的埋设细节 | Grep 搜索关键词 | 搜索"绿灯笼"找到第1章的原始描述 |
| 确认角色某次出场的对话 | Read 对应章节文件 | 读取 015-锚的温度.md 获取苏黎的台词 |
| 确认地点描写的一致性 | Grep 搜索地点名 | 搜索"城隍庙"获取所有相关描写 |

**关键原则**：正文是「只读参考资料」，写作时只参考不修改（除非用户明确要求修改早期章节）。

## 五、写作流程标准操作程序（SOP）

### 5.1 SOP-A：「续写 N 章」完整工作流

**触发条件**：用户说「续写3章」「继续写」等

```
步骤 1：信息收集（主智能体）
├── 读取 11-进度追踪.md → 确认当前写到哪
├── 读取 04-卷纲设计.md（当前卷）→ 确认大方向
├── 读取最近 5 章正文 → 把握当前叙事状态
├── 读取 08-伏笔追踪表.md → 确认待回温/待回收的伏笔
└── 读取 02-角色设定.md（相关角色）→ 确认角色行为一致

步骤 2：剧情规划（planner 子智能体）
├── 输入：上下文包（见第二部分 2.3）
├── 输出：逐章大纲，每章包含：
│   ├── 章节标题
│   ├── 情绪目标（1-2个关键词）
│   ├── 核心冲突（一句话）
│   ├── 段落拆解（3-5段，每段含字数、内容、技法标注）
│   ├── 章尾钩子类型（悬念/危机/情感/反转/信息）
│   ├── 代价变化（如有）
│   └── 新增/回收伏笔编号
└── 技法校准：对照 05-叙事技法规划 的节奏公式

步骤 3：用户确认（主智能体）
├── 将逐章大纲展示给用户
├── 收集修改意见
└── 定稿

步骤 4：正文写作（主智能体，逐章执行）
├── 每章写作前再次确认：
│   ├── 本章的伏笔任务（埋设/回温/回收）
│   ├── 上一章的结尾状态（衔接点）
│   └── 角色当前的情绪和能力状态
├── 写作过程中的自检项：
│   ├── 每章 3500-4500 字
│   ├── 至少一个微型爽点或信息推进
│   ├── 章尾必须使用断章技巧
│   ├── 代价描写遵循"具体可感"原则
│   └── 对话要有角色辨识度（不能所有人说话一个味）
├── 写完每章后：
│   ├── 保存为 `正文/卷X-卷名/{章编号}-{标题}.md`
│   └── 进入步骤 5

步骤 5：质量审查（code-reviewer 子智能体，每 3-5 章批量执行）
├── 检查项：
│   ├── 角色行为一致性（对比 02-角色设定）
│   ├── 伏笔使用是否到位（对比 08-伏笔追踪表）
│   ├── 世界观是否有矛盾（对比 01-世界观设定）
│   ├── 断章效果是否有效
│   ├── 文风是否统一（恐怖/燃刀/日常场景的语气切换）
│   └── 章节字数是否合理
├── 输出：审查报告（通过/需修改+具体建议）
└── 如需修改 → 主智能体执行修改

步骤 6：文档更新（主智能体）
├── 更新 08-伏笔追踪表.md
│   ├── 新增伏笔：分配 F/H 编号
│   ├── 状态变更：🔴→🟡 或 🟡→🟢
│   └── 金句记录
├── 更新 11-进度追踪.md
│   ├── 当前章节、字数
│   ├── 共鸣度变化
│   └── 代价清单
└── 如有新角色 → 更新 02-角色设定.md
```

## 六、《斩神》等优秀网络小说写法分析

> 基于写作教程 17-analysis-slash-god.md、16-analysis-horror-house.md、18-analysis-mysterious-revival.md 的分析结论，提炼可直接应用于《人间镇陵人》的具体技法。

### 6.1 世界观宏大但起始微小的写法

**《斩神》的范例**：

斩神的世界观涵盖多国神话体系、守夜人组织、界碑系统等宏大设定，但开篇只聚焦在一个精神病院里的失明少年身上。读者的体验路径是：

```
精神病院（小）→ 守夜人训练基地（中）→ 大夏全境（大）→ 国际神话碰撞（巨）
```

**迁移到《人间镇陵人》**：

```
大学宿舍（小）→ 溟渊市老城区（中）→ 华东六城（大）→ 十座神陵全境（巨）
```

**具体执行要点**：

| 原则 | 斩神做法 | 本作对应做法 |
|------|---------|-------------|
| 前5章只展示"冰山一角" | 只暗示"月亮上有人"，不解释守夜人 | 只暗示"护身符会发光"，不解释神陵 |
| 世界观通过角色经历自然展开 | 林七夜被带入守夜人，读者跟着他学 | 沈夜被韩铮引入镇陵司，读者跟着他了解 |
| 每次扩展只增加一层新信息 | 先学禁墟，再学界碑，再学多神体系 | 先学共鸣度，再学神陵，再学十神体系 |
| 大格局元素作为"远处的雷声" | 偶尔提及国际局势，但主角暂时接触不到 | 偶尔提及蚀门门主、其他神陵，但沈夜暂时管不到 |

**检查标准**：前30章（约10万字），读者应该知道以下信息存在，但不完全理解：
- 十座神陵的存在（只知道太昊陵一个）
- 镇陵司的全国体系（只接触了溟渊站）
- 蚀门的完整层级（只见过呓一个祭司）
- 共鸣度的五个阶段（只经历了觉醒和侵蚀）

### 6.2 子副本不平铺直入的技巧

**《恐怖屋》的范例**：

恐怖屋的每个"鬼屋副本"不是简单地"进去→吓人→出来"，而是有"探查→发现规则→利用规则→意外→解决"的五步结构。关键是**每个副本都有独特的"规则"**，读者会期待"这次的规则是什么"。

**迁移到《人间镇陵人》——灵异事件设计**：

本作的灵异事件有三种类型（规则型/战斗型/契约型），每种类型的子副本设计遵循以下模板：

```
灵异事件子副本结构：

1. 异常感知（10%）
   → 沈夜感知到"有东西不对"，但不确定是什么
   → 读者跟着他一起紧张

2. 规则发现（25%）
   → 通过调查/观察，发现这个灵异事件的"规则"
   → 规则型：某种空间/记忆/时间规则
   → 战斗型：灵体的弱点和攻击模式
   → 契约型：交易的条款和隐藏代价

3. 规则利用/对抗（30%）
   → 沈夜尝试利用规则解决问题
   → 但总是有"意外"——规则比预想的更复杂

4. 代价爆发（15%）
   → 解决问题的同时，共鸣度上升
   → 一次具体的代价落在某个羁绊上

5. 余波与新线索（20%）
   → 事件解决，但留下了新的线索或悬念
   → 为下一个事件或主线推进铺路
```

**不平铺直入的三个技巧**：

| 技巧 | 说明 | 本作示例 |
|------|------|---------|
| **入口错位** | 灵异事件的"入口"不在你以为是的地方 | 城隍庙的灵异不在庙里，而在铜镜中（第12章） |
| **规则嵌套** | 表面有一层规则，深处还有一层 | "门消失"事件的表面规则是空间错位，深层是呓在测试沈夜（第20章） |
| **角色反套** | 看起来是NPC的角色其实是关键线索 | 马三爷看起来是搞笑反派，实际是蚀门的监控棋子（第14章） |

### 6.3 男频群像的叙事节奏

**《斩神》的范例——标签化+反差+成长线三层结构**：

每个群像角色三件套：标签（让人记住）+ 反差（让人惊喜）+ 成长线（让人牵挂）。

**迁移到《人间镇陵人》**：

| 角色 | 标签 | 反差 | 成长线 |
|------|------|------|--------|
| **陆渊** | 缺指的神秘古董商 | 对鬼不怕但对人沉默 | 观察者→参与者→传承者 |
| **顾沉** | 琥珀色眼眸的世家公子 | 桀骜但渴望被认可 | 竞争→惺惺相惜→殊途同归 |
| **陈九** | 190cm的"快递员"打手 | 粗犷但从不在我面前抽烟 | 互看顺眼→默契→兄弟 |
| **苏黎** | 体温温暖的医学生 | 温和但极其有主见 | 守护→介入→并肩 |

**群像戏份分配原则**：

```
每 10 章一个"群像周期"：
├── 主线推进（沈夜视角）：5-6 章
├── 配角高光（某位配角的核心场景）：2-3 章
├── 群体互动（多人场景/战斗）：1-2 章
└── 暗线推进（周远/蚀门/太昊的视角）：0.5-1 章（穿插在主线中）
```

**群像叙事的黄金法则**：
- 大场景用小切口——从某个角色的视角切入，而非"上帝视角扫一遍"
- 宏大叙事中插入个人细节——一个眼神、一句话、一个回忆
- 牺牲要有"具体的代价"——不是"死了很多人"，而是"XX再也见不到他的女儿了"

### 6.4 其他参考作品的可用技法

**《神秘复苏》——规则推理型恐怖**：
- 每个灵异事件都有"规则"，角色必须通过推理找到规则
- 本作应用：神陵的灵异事件也遵循"规则"——太昊陵的空间规则、沧溟陵的记忆规则
- 技法：让读者参与推理——给出足够线索，让聪明的读者在主角之前猜出规则

**《恐怖屋》——经营+恐怖双线并行**：
- 两套完全不同的叙事模式交替出现
- 本作应用：镇陵司的"工作日常"（组织管理、训练、情报收集）+ 灵异事件的"恐怖模式"交替
- 技法：日常越是正常温馨，灵异就越恐怖——反差制造张力

**《十日终焉》——高智商破局**：
- 主角不是靠蛮力，而是靠智慧和策略解决问题
- 本作应用：沈夜的"数三秒"仪式体现了理性克制；在神陵探索中需要解谜而非硬闯
- 技法：让读者看到主角的思考过程，而不只是看到结果

## 七、针对《人间镇陵人》特点的定制建议

### 7.1 "有刀有燃"的节奏设计

本作的情感基调是"克制下的燃烧"——与《斩神》的纯粹热血不同，本作的燃总伴随着痛。

**本作特有的燃刀一体公式**：

```
燃刀一体 = 力量爆发的震撼 + 代价落下的心痛
```

具体执行模式（基于已有前 25 章的成功案例）：

| 模式 | 结构 | 已有案例 | 后续规划 |
|------|------|---------|---------|
| **先燃后刀** | 爽了→然后发现代价 | 第13章：净化使徒→忘记食物 | 卷四：沈渊灭神术壮烈→化光融入封印 |
| **刀中见燃** | 代价的瞬间反而看到了人性的光辉 | 第17章：忘记最后一顿饭→铜镜中母亲的脸 | 卷五：保护所有人的燃→叫不出任何人的名字 |
| **燃就是刀** | 越燃越痛，读者又想看又不忍看 | 第25章：太昊问答的释然→墓门新增"我在等你" | 卷六终章：四势力联手的壮烈→门主化为光尘 |

**燃刀频率控制**：

```
小周期：2章小燃 + 1章小刀 = 3章一个呼吸
大周期：2个完整小周期 → 1章大燃或大刀 = 约7章一个高潮
每卷：3-4个大周期

全年节奏：
卷一(铺垫) → 卷二(燃+刀) → 卷三(群像燃) → 卷四(大燃+大刀) → 卷五(终极燃) → 卷六(燃刀一体)
```

### 7.2 "智力在线"的剧情设计要求

本作主角沈夜不是莽夫，他的核心行为模式是"数三秒"——在行动前给自己思考的空间。这一设定要求所有剧情必须经得起推敲。

**智力在线的四条写作纪律**：

**纪律一：灵异事件必须有逻辑可循**

每个灵异事件都有"规则"，沈夜的解决方式必须是"发现规则→利用规则"，而非"蛮力硬刚"。

```
反面示范：沈夜冲上去用净化之力把灵体消灭了
正面示范：沈夜观察灵体的行动规律，发现它只在"有镜面反射"的地方出现，
         于是打破所有镜子，灵体失去锚点后被净化
```

**纪律二：角色不能为剧情降智**

不能为了让剧情推进而让角色做出不符合其设定的蠢事。

| 角色 | 不能做的事 | 因为 |
|------|-----------|------|
| 沈夜 | 明知代价还无脑用神力 | 他的核心设定就是"克制" |
| 陆渊 | 无理由地隐瞒关键信息 | 他已经承诺"该告诉你的不再隐瞒" |
| 顾沉 | 不计后果地冲动 | 他是"控制欲强"的人，控制欲意味着计划性 |
| 蚀门 | 低级失误 | 作为隐藏千年的组织，必须保持威胁感 |

**纪律三：伏笔的埋设和回收要自然**

```
自然的伏笔：读者事后回忆时觉得"原来那时候就有暗示了"
不自然的伏笔：角色突然说了一段完全没来由的话，显然是为了后面埋线
```

**纪律四：反派要有合理的行为逻辑**

- 蚀门的每一步行动都有明确目的（不是"为了当反派而做坏事"）
- 呓的每一次出现都有战略考量（试探、施压、收集数据）
- 中距反派裴行宜的立场"不算错"，只是方式有问题
- 终极反派太昊甚至不是"恶"的——他只是想"纠正错误"

### 7.3 群像戏份分配原则

本作的群像不是《斩神》式的"标签化功能型群像"，而是"命运纠葛型群像"。每个核心配角都与主角有"因果纠缠"。

**三层群像管理**：

| 层级 | 角色 | 戏份占比 | 功能 |
|------|------|---------|------|
| **S级（2人）** | 陆渊、顾沉 | 各占 8-12% | 提供叙事深度和对抗张力 |
| **A级（3-5人）** | 苏黎、陈九、沈昭、顾霜、韩铮 | 各占 3-8% | 提供情感支撑和剧情引擎 |
| **B级（5-10人）** | 赵令仪、周远、马三爷、钱眼等 | 各占 1-3% | 提供世界观丰富度和功能支撑 |

**戏份分配的"呼吸节奏"**：

```
卷一：主角70% + 陆渊10% + 苏黎10% + 其他10%
卷二：主角60% + 陆渊10% + 顾沉15% + 其他15%
卷三：主角50% + 群像各10%×5 = 50%（群像爆发卷）
卷四：主角55% + 沈渊15% + 母亲线15% + 其他15%
卷五：主角50% + 顾沉15% + 反派线15% + 群像20%
卷六：主角45% + 所有核心角色各占一份（终章大汇演）
```

**群像出彩的关键**：

不是给配角更多戏份就等于好群像。关键是每个配角在他的戏份里要有**自己的选择**——不是"帮主角做事"，而是"做自己的事，恰好与主角相关"。

| 配角 | 他的"自己的事" | 与主角的交汇点 |
|------|---------------|---------------|
| 陆渊 | 赎1937年的罪 | 他选择的赎罪方式恰好是保护沈夜 |
| 顾沉 | 救妹妹顾霜 | 救妹妹需要太昊碎片→需要沈夜配合 |
| 陈九 | 不丢失自己的记忆 | 保护沈夜是他保持记忆的方式 |
| 苏黎 | 理解"看不见的世界" | 她的好奇心推动她进入沈夜的世界 |
| 周远 | 保护沈夜（即使被控制） | "你妈妈让我保护你"是他的真实意志 |

### 7.4 "羁绊代价"的叙事红线

这是本作最核心的独特设定，必须严格保护。

**代价描写的四条红线**：

1. **每次代价必须具体可感**：不是"失去一些感情"，而是"酒窝的位置变成空白"
2. **代价只进不退**：已失去的记忆不可恢复（除非剧情重大转折）
3. **代价分散落地**：不要所有代价都集中在苏黎身上，分散到母亲、周远、陆渊
4. **代价要有递进**：从表情细节→场景氛围→完整人物→整段关系，越往后越痛

**代价节奏的"呼吸"设计**：

```
无代价的章节 → 微代价（细节模糊）→ 无代价 → 中代价（完整记忆消失）→ 无代价 → 无代价 → 大代价（整个人的记忆开始瓦解）
```

不能连续2章以上出现代价，否则读者会麻木。代价之间必须有"糖"来对比——苏黎的温暖、陆渊的保护、陈九的幽默，都是为下一次代价蓄力。
