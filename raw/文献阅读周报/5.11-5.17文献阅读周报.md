**第一篇论文：Beyond Self-Talk: A Communication-Centric Survey of LLM-Based Multi-Agent Systems**

这篇论文是 2025 年 2 月公开在 arXiv 上的一篇综述。它的视角很明确，不再像以往很多综述那样主要按应用或者整体架构来整理多智能体系统，而是把“通信”单独拉出来，作为理解 LLM-based MAS 的核心切口。作者认为，多智能体系统是否有效，很多时候不只是取决于单个 agent 本身的能力，更取决于 agent 之间如何交流、如何协商以及如何组织协作。

论文的具体内容主要分成两个层次。第一层是 system-level communication，讨论整个系统层面的通信组织方式，包括架构、通信目标和通信协议；第二层是 system-internal communication，讨论具体交互时 agent 怎么说、对谁说、传递什么内容，进一步细分为 communication strategy、communication paradigm、communication object 和 communication content。也就是说，这篇论文最重要的贡献不是提出新方法，而是给多智能体通信问题提供了一个比较完整的分析框架。

作者指出的未来挑战主要集中在几个方面：一是通信效率问题，频繁交互会带来额外开销；二是通信协议仍不统一，缺少通用标准；三是随着 agent 数量增加，系统的可扩展性和稳定性会迅速下降；四是通信链路中还存在安全风险、错误传播和 benchmark 不足的问题。整体来看，这篇综述更适合作为“多智能体通信”这一方向的入门材料。

**第二篇论文：LLMs Working in Harmony: A Survey on the Technological Aspects of Building Effective LLM-Based Multi Agent Systems**

这篇论文是 2025 年的 arXiv 综述。和第一篇不同，它的视角不是从通信切入，而是从“如何真正把多智能体系统搭起来”这个问题出发，更强调技术实现层面的分析。作者想回答的是：如果要构建一个有效的 LLM-based multi-agent system，那么最关键的技术支柱到底有哪些。

论文把多智能体系统拆成四个主要维度来讨论，分别是 architecture、planning、memory 和 technologies/frameworks。围绕这四个维度，作者系统梳理了不同架构组织方式、规划策略、记忆机制以及工程框架。例如在 architecture 部分，讨论多 agent 如何组织与分工；在 planning 部分，讨论系统如何推理、决策和执行；在 memory 部分，讨论短期记忆、长期记忆以及检索增强记忆的差异；在 frameworks 部分，则整理了 AutoGen、CAMEL、MetaGPT、LangGraph 等代表性框架。整篇论文更像一张技术地图，而不是单纯的工作罗列。

作者指出的未来挑战主要包括：第一，虽然技术维度已经比较清楚，但不同方法之间仍缺少统一评测，很多结论难以直接横向比较；第二，随着多智能体系统越来越复杂，架构、规划、记忆之间的耦合问题会越来越明显；第三，现有框架虽然丰富，但还没有形成足够稳定、通用的工程范式。总体来看，这篇综述更适合帮助我建立“多智能体系统到底由哪些技术部分构成”的整体认识。

**第三篇论文：Creativity in LLM-based Multi-Agent Systems: A Survey**

这篇论文发表于 EMNLP 2025，是一篇正式会议综述。它的视角和前两篇又不一样，不是从通信或技术支柱出发，而是专门聚焦“创造力”这个维度。作者想讨论的是：多个大模型 agent 协同工作时，是否能够产生比单个 agent 更有创造性的结果，以及这种创造力究竟来自哪些机制。

论文的具体内容主要围绕四个方面展开。第一是 workflow 和 proactivity，讨论创意生成的工作流如何组织，以及 agent 是被动响应还是主动提出想法；第二是 techniques，重点整理了 divergent exploration、iterative refinement 和 collaborative synthesis 这几类创造性生成机制；第三是 persona and profile，分析不同角色设定和人格粒度对创意输出的影响；第四是 evaluation，讨论创造力应该如何评估，包括客观指标、主观评价以及交互式评价方式。也就是说，这篇综述的重点不在于泛泛谈“多智能体有创造力”，而在于把创造力拆成可分析、可设计、可评测的问题。

作者指出的未来挑战主要有三点。第一，创造力的评测标准仍不统一，不同工作之间难以公平比较；第二，创造力本身带有很强的主观性，不同任务和不同文化背景下的评价可能差异很大；第三，多智能体创造过程还会遇到角色冲突、偏见积累和协作失衡等问题。总体来看，这篇论文更适合作为理解“多智能体创造性协作”这一方向的综述入口。

**第四篇论文：G²CP: A Graph-Grounded Communication Protocol for Verifiable and Efficient Multi-Agent Reasoning**

这篇论文不是综述，而是一篇具体的方法论文。它先在 2026 年 2 月以 arXiv 预印本形式公开，随后发表于 AAMAS 2026。它要解决的问题很直接：现在很多多智能体系统仍然依赖自然语言进行 agent 间通信，但自然语言通信容易产生语义漂移、幻觉传播以及额外的 token 开销，尤其在多轮协作推理时，这些问题会被不断放大。

作者提出的方法是 G²CP，也就是 Graph-Grounded Communication Protocol。它的核心思路是：不再让 agent 用自由文本彼此交流，而是改用结构化的图操作作为通信语言。具体来说，agent 之间交换的是对共享知识图谱的遍历、查询和更新操作，而不是自然语言消息。这样做的目的，是把通信语义锚定在图结构上，让推理过程更明确，也更容易审计和复现。

这篇论文最值得注意的是它对“多智能体通信协议”做了结构化改造。作者把不同 agent 设计成围绕共享知识图谱执行不同功能的角色，而 LLM 主要负责用户问题理解和最终回答生成。也就是说，文本生成能力和结构化协作能力被分开处理，这一点和很多纯 prompt 驱动的多智能体工作很不一样。

这篇论文也暴露出一些后续值得关注的问题。第一，这种方法高度依赖知识图谱，图谱构建和维护成本很高；第二，如果任务依赖大量图外知识，图协议的优势会受到限制；第三，当前实验主要集中在工业维护场景，跨领域泛化能力还有待进一步验证。整体来看，这篇论文让我意识到，如果以后想真正解决多智能体中的幻觉和错误传播问题，通信协议层面的结构化设计可能会是一个很重要的方向。
