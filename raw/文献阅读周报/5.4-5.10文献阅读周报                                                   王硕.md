**第一篇论文：Large Language Model based Multi-Agents: A Survey of Progress and Challenges**

该篇论文核心想要解决的问题是：
1. **Agent和环境是如何交互的？**
   通过 智能体——环境接口，环境接口被分为sandbox、physical、none
2. **一个多智能体系统里的 agent 是怎么“设定身份”的？**
   智能体画像：特征、行动、技能、约束。而智能体画像定义方式分为：人工预定义、模型自动生成、从已有数据中提取
3. **agent 之间是怎么沟通和协作的？**
   作者将通信分为三层：1、通信范式：协作、竞争、辩论。 2、通信结构：中心化、去中心化、分层、共享消息池  3、通信内容：以文本为主
4. **这些 agent 的能力是怎么增长、怎么调整的？**
   能力提升来源于两部分：1、反馈调整：可以由环境、智能体评价、人类的反馈进行提升。  2、靠记忆、反思、自我演化或者动态生成新agent来变强

**核心内容**：论文将多智能体系统拆解为四个维度：**环境接口、画像设定、通信、能力获取**，并且将当时的应用归纳成两条主线：**面向问题求解** 与 **面向世界模拟**。论文还总结了工具、框架、数据集和benchmark。
其中，框架有MetaGPT(更强调**流程化、角色分工、SOP**)、CAMEL（更强调**角色扮演与对话协作**）AutoGen（更强调**可编程性和灵活 agent 交互**）
benchmark做接的结论是：目前还没有统一的benchmark.

**挑战**：
1、Multi-modal environment：目前多文本，可考虑转多模态
2、Hallucination：级联幻觉
3、Scaling up：规模导致的成本与协调困难
4、collective intelligence：真正的群体智能还有距离

这篇论文是2024年的综述论文，相对于多智能体领域的飞速发展显得不够深入与前沿。



**第二篇论文：A Survey on LLM-based Multi-Agent System: Recent Advances and New Frontiers in Application**

该篇论文更强调应用前沿、资源整理和评测问题。

**核心内容**：将基于大语言模型的多智能体系统的应用分为三类：
1. **任务求解（task-solving）**
2. **面向特定问题的模拟（simulation for specific problems）**
3. **对生成式智能体的评估（evaluation of generative agents）**
作者将LLM-MAS压缩成两个核心部件：**生成式智能体**和**环境**
生成式智能体需要具备：画像设定、记忆、规划。行动
环境构成：规则、工具、干预接口

**挑战**：本论文将挑战分为三层：
1、智能体自身（对齐/幻觉/长文本）
2、交互过程（效率爆炸/累积效应）
3、评测（群体指标缺失/benchmark 缺失）

该篇论文更聚焦于应用的分类，并且对于资源的整理更加深入，**作者分析了开源框架，以及带有 benchmark 或 dataset 的研究工作，以帮助研究社区更快进入该方向**


**第三篇论文：Multi-Agent Collaboration Mechanisms: A Survey of LLMs**

该篇论文聚焦于多智能体系统的**协作层面**，作者指出：高效 MAS 的核心关注点之一，就是**协作机制**。AI 正在从传统孤立模型转向强调交互的范式：agent 需要连接、协商、做决策、规划并联合行动。因此，更深入地理解协作机制如何运行，是释放 MAS 潜力的关键。
该论文的**协作系统定义与形式化**我认为很有意思。通过符号化的形式来表征多智能体系统。
1. 单个 agent 被形式化为 `model + objective + environment + input + output`。
2. 多智能体系统真正新增的核心对象，不只是 `多个 agent`，而是 `collaboration channels`。
3. 作者把协作看成一种系统级机制，而不只是“互相发消息”。
4. 只要 actors / types / structures / strategies 不同，就可以视为不同协作通道。

作者提到**working together 不只是 communication（信息交换），而是更深层的 coordination and management。**

作者在讲述完协作系统的定义与形式化后，又从下述 个方面拆解多智能体系统的协作层面：
1、协作类型：协作、竞争、竞合、混合
2、协作策略：规则驱动、角色驱动、模型驱动
3、通信结构：Centralized、Decentralized 、Distributed、Hierarchical
4、协调编排：Static architecture、Dynamic architecture
5、协调编排的经验总结：
1. **Effective Collaboration Channels**
   建立清晰可靠的协作通道极其关键。好的 collaboration mechanism 能让 MAS 超过单 agent；差的设计反而会输给单 agent。
2. **Collective Domain Knowledge**
   领域知识对于设计协作架构和 crafting system prompts 很重要。很多 channel 都是围绕领域知识显式预定义的。
3. **Adaptive Role and Channel Assignment**
   某些任务中，让系统动态分配角色和 channel 比固定分配更灵活更有效。
4. **Optimal Collaborative Strategy**
   - 规则驱动适合高一致性、强规范任务；
   - 角色驱动适合工作分工明确的任务；
   - 模型驱动适合不确定、动态环境。
3. **Scalability Considerations**
   agent 数量增加后，协调复杂度显著上升，需要更可扩展的架构与算法。
4. **Ethical and Safety Considerations**
   agent 必须在伦理边界内行动，否则协作系统会放大错误与风险。

**挑战**：该论文将挑战分为多个领域的不同方向：
1. **通往人工集体智能之路**：Unified Governance、Shared Decision Making、Agent as Digital Species、Scalability and Resource Maintenance、Discovering / Exploring Unexpected Generalization
2. **全面评测与 benchmark**
3. **伦理风险与安全**

这篇论文对多智能体系统的协作层面的分析的较为全面。
