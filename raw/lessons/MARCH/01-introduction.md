# MARCH — 1. 引言 (Introduction)

> LLM 幻觉是 RAG 系统的核心瓶颈。现有验证方法受困于确认偏误，MARCH 通过信息不对称的三智能体强化学习框架从根本上解决这一问题。

## 正文翻译

### 1 Introduction

尽管大语言模型（LLMs）取得了显著成功，幻觉（hallucinations）的持续存在仍然是 LLM 可信使用的基本障碍。即使在检索增强生成（RAG）框架内加以增强，LLM 仍然频繁生成在语言上流畅但与提供的证据上下文相矛盾的回答。这种上下文冲突型幻觉在数据密集型任务中尤为普遍，表现为不准确的数字、扭曲的时间序列或错误的算术汇总——即使源文本中明确包含所需信息。在高风险领域（如金融、法律和医疗），幻觉更具破坏性，因为用户需要 LLM 充当细致的证据阅读者，而非看似合理但未经验证的叙述的创造者。

> [!note] 译者注（非原文）
> "上下文冲突型幻觉"（context-conflicting hallucinations）是幻觉分类中的一种类型，指模型生成的回答与检索到的文档内容直接矛盾。与之相对的还有"内在冲突型"（自相矛盾）和"外在冲突型"（与已知世界知识矛盾）。

在缓解幻觉的方法中，监督微调（SFT）被认为是一种有效的手段。然而，多项实践表明，SFT 也可能放大幻觉，因为它本质上是在进行行为克隆，这会诱导模型优先模仿权威风格模式，而非追求事实准确性。

因此，强化学习（RL）已成为优化 LLM 向更真实性方向发展的必要手段。然而，标准的结果级 RL 奖励缺乏监督细粒度事实一致性所需的精度，导致标量奖励与 RAG 细致证据要求之间存在根本性错位。

具体而言，RLHF 中的奖励模型通常将多维度评估压缩为粗粒度的标量分数。这种粒度损失限制了对单个声明进行验证的能力，无法提供足够的信号来强化数据密集型任务所需的细致证据锚定。另一方面，虽然 RLVR（带可验证奖励的强化学习）尝试通过确定性规则、评分准则或外部通用奖励模型提供客观反馈，但它仍受制于双重依赖瓶颈：专业领域专家标注真值的稀缺性，以及外部验证器的推理上限。因此，未经验证的中间声明可能持续存在，因为模型仅因正确的最终结果而获得奖励，尽管推理轨迹中可能包含幻觉。

虽然事后度量和事实检查器在评估中提供了实用价值，但此类静态工具很少被整合到内部优化循环中以提供可操作的、面向声明的指导。更关键的是，此类验证器的有效性从根本上受到当前评估方案中固有的信息泄漏的破坏。具体而言，同时接触查询、文档和生成回答会诱发确认偏误。在这种状态下，验证器倾向于优先验证内部一致性，而非基于源证据的客观锚定，导致对错误声明的不当认可。这种妥协的"盲审"阻止了模型内化高保真生成所必需的严格、文档锚定的推理。

> [!tip] 理解提示（非原文）
> 这里点出了论文的核心洞察：现有验证方案的问题不在于验证器能力不足，而在于**信息架构有缺陷**——验证器同时看到了生成结果和源文档，自然会倾向于"自圆其说"而非真正交叉检验。

为解决上述事实性挑战，我们引入了多智能体强化自检（Multi-Agent Reinforced Self-Check for Hallucination, MARCH），一个基于 RL 的框架，通过刻意的信息不对称来强制事实对齐。

MARCH 编排了一条由三个专门智能体组成的协作流水线：Solver、Proposer 和 Checker，以防止基于模型的验证器通过确认偏误走认知捷径。

如图 1 所示，Solver 首先基于查询和检索文档生成回答。随后，Proposer 作为"回答原子化器"将叙述拆解为离散的可验证声明。最后，Checker 通过仅基于源文档重新回答来验证这些声明，严格对 Solver 的原始输出保持盲态，以避免认知污染。通过这种不对称解耦，MARCH 将验证过程从对合理文本的循环确认转变为生成声明与事实证据之间的严格交叉检验。

在训练过程中，MARCH 通过零容忍奖励机制促进智能体自我进化：Proposer 提取的声明与 Checker 基于文档的验证之间的任何不一致都会触发对整条响应轨迹的惩罚。这鼓励智能体严格将输出与检索证据对齐，有效地内化严格、文档锚定陈述的能力。

实验上，MARCH 在基座模型基础上实现了幻觉的显著减少，无需额外人工标注或外部事实检查工具，从而证明了我们不对称协作强化范式的有效性。值得注意的是，配备 MARCH 的 8B 参数 LLM 在多个幻觉基准测试上达到了与领先闭源模型的性能持平，同时在通用 RAG-QA 任务上也取得了显著增益。这些结果为 LLM 的可信、智能体自我改进建立了一条可扩展且可验证的路径。

> [!warning] 易混点（非原文）
> "信息不对称"（Information Asymmetry）在经济学中通常指交易双方拥有不同信息。本文借用此概念，但含义不同：指三个智能体之间的**刻意信息隔离**——Checker 看不到 Solver 的输出，打破验证器与生成器之间的信息共享，从而消除确认偏误。这与"信息瓶颈"或"信息丢弃"也不同，因为信息并未被丢弃，而是被隔离到不同的智能体角色中。

## 关键要点

1. **LLM 幻觉的根本挑战**：即使有 RAG 增强和 RLHF 对齐，幻觉仍然存在且难以消除
2. **确认偏误是核心瓶颈**：现有验证器同时看到生成结果和源文档，导致循环确认而非交叉检验
3. **MARCH 的核心思想**：通过信息不对称的三智能体分工，将验证转化为严格的交叉检验
4. **零容忍奖励**：任何单个声明不匹配即惩罚整条轨迹，强制模型追求完全的事实准确性
5. **8B 模型匹配闭源模型**：无需额外标注或外部工具，纯靠自博弈实现事实对齐
