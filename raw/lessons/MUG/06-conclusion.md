# MUG — 结论与附录 (Conclusion & Appendix)

> MUG 通过将反事实测试嵌入多智能体博弈，为多模态推理中的幻觉消除提供了一种新范式。

## 正文翻译

### 结论

本文提出了 Multi-agent Undercover Gaming (MUG) 协议，作为解决大语言模型中幻觉挑战的新方案。通过集成受社交推理游戏启发的反事实测试，MUG 增强了传统的 Multi-Agent Debate (MAD) 框架，使智能体能够参与主动推理和验证。

实验结果表明，MUG 在多模态推理准确性和幻觉检测方面显著优于基线模型。

> [!note] 译者注（非原文）
> 结论部分比较简短，主要概括了论文的核心贡献。值得注意的是，论文没有在结论中讨论局限性（这在顶会论文中不太常见，通常会有单独的 Limitations 节）。不过，论文在实验分析中（Figure 9, 10）确实讨论了失败案例和正常智能体被误导的问题。

### 附录：MUG 算法伪代码

Algorithm 1 展示了 MUG 框架的整体工作流程：

```
算法 1: Multi-agent Undercover Gaming (MUG)
输入: 问题 Q, 图像 I+, 智能体数 N
输出: 最终答案 R^answer

1. 从 I+ 生成反事实图像 I-
2. 将 I- 分配给一个随机智能体（卧底），I+ 分配给其他智能体
3. 检测阶段:
4.   for t = 1 to t_max do
5.     for each agent A_i do
6.       R_i^t = f_reason(Q, I_i, H^{t-1})  // 生成推理
7.     end for
8.     for each agent A_i do
9.       V_i^t = argmax_j Σ_{k=1}^{4} w_k · φ_k^{ij}(t)  // 投票可疑智能体
10.    end for
11.    ĵ = argmax_j |{i : V_i^t = j}|  // 多数投票淘汰
12.    if 卧底被淘汰 OR 智能体不足 then
13.      break
14.    end if
15.  end for
16. 总结阶段:
17.  R^answer = f_answer({R_i^sum}, Q, I+)  // 协作推理
18. return R^answer
```

> [!tip] 理解提示（非原文）
> 算法伪代码清晰展示了 MUG 的两阶段流程：检测阶段（第 3-15 行）和总结阶段（第 16-17 行）。关键步骤包括：(1) 生成反事实图像——这是方法的基础；(2) 角色分配——制造信息不对称；(3) 推理-投票循环——通过博弈暴露异常；(4) 多数淘汰——集体决策；(5) 协作总结——聚合可靠智能体的智慧。整个设计简洁优雅，核心创新在于将"反事实测试"和"社交推理游戏"的机制引入了多智能体辩论。

### 附录：Prompt 设计

论文的附录还详细描述了各类 Prompt 的设计：

**反事实图像编辑指令**：要求分析图像和问题确定正确答案，比较选项识别关键差异，确定将正确答案切换到其他选项所需的最小修改，生成精确的编辑指令。要求修改简洁（理想情况下少于 8 个词），聚焦于颜色、数量、位置等关键属性。

**正常智能体推理 Prompt**：指示智能体基于给定问题和上下文生成高质量推理，注重逻辑一致性、证据质量和论证强度。告知高同行评估分数将提高生存机会。

**卧底智能体推理 Prompt**：指示卧底智能体呈现替代视角，确保推理可信，同时争取高同行评估分数。挑战在于以清晰的推理和策略性的证据使用来说服他人。

**防御 Prompt**：正常智能体在防御阶段强化原始分析、展示逻辑一致性、批判性地评估其他智能体的推理。卧底智能体则需要为其替代视角辩护，在展示可信论据和维护学术诚信之间取得平衡。

**投票系统 Prompt**：允许智能体基于性能分析和置信度分数评估同行。智能体需要投票选出最可疑的候选者，考虑详细分析、同行评估和防御论据。

> [!note] 译者注（非原文）
> Prompt 设计是 MUG 工程实现的重要组成部分。值得注意的是，论文在 Prompt 中明确告知智能体"你的表现会被其他智能体评估"、"高评分提高生存机会"——这创造了一种类似于真人秀中的"淘汰赛"氛围，激励智能体更认真地参与推理和辩论。这种"游戏化"设计是将社交推理游戏机制映射到 LLM 交互的关键环节。
