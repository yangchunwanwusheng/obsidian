# GUARDIAN — 引言 (Introduction)

> 本节介绍多智能体协作面临的安全挑战，以及 GUARDIAN 的核心思路。

## 正文翻译

### 背景与动机

大语言模型 (LLM) 的出现彻底改变了人工智能领域，使得能够进行复杂、多轮对话的智能体开发成为可能 [1-5]。这些基于 LLM 的智能体在各种任务中展现了卓越的能力，从问答和信息检索到问题求解 [6-8]。多智能体协作系统 [9, 10] 使多个 LLM 智能体能够通过交互式通信和协作推理共同解决复杂问题 [11]。

多智能体协作框架的兴起，特别是通过新兴的 Agent2Agent (A2A) [12] 协议，为 AI 协调提供了新的范式。虽然这些系统在增强 AI 能力方面具有巨大潜力，但也引入了独特的挑战和安全问题。这些关键安全问题包括：

- **智能体间的幻觉传播** [9, 13-18]：一个智能体生成的非事实信息通过交互网络传播并放大。
- **错误被恶意注入和传播** [17, 19-23]：恶意行为者注入错误，导致先前可靠的智能体采纳并持续传播这些错误。

因此，本文将这些安全问题分为两类：
1. **幻觉放大 (Hallucination Amplification)**——不涉及人为因素
2. **错误注入与传播 (Error Injection and Propagation)**——由人为因素造成

> [!tip] 理解提示（非原文）
> "人为因素"的含义：幻觉放大是 LLM 自身生成不准确内容后在协作中自然扩散，而错误注入是有恶意攻击者主动操控智能体或通信渠道。

注入的错误可进一步分为：
- **智能体定向攻击 (Agent-targeted attacks)**：通过恶意系统提示词 (system prompts) 破坏智能体 [24]
- **通信定向攻击 (Communication-targeted attacks)**：在对话轮次之间破坏信息流 [22, 25]

Figure 1 展示了这些关键安全问题，说明了幻觉和注入错误如何在多智能体系统中传播。这些问题对基于 LLM 的多智能体系统构成重大风险，需要稳健的检测和缓解方法。

### 现有防御方法的局限

针对 LLM 智能体协作中幻觉和错误传播的防御措施可分为两类：

**协作式错误检测方法**，如交叉审查 (cross-examination) [26] 和外部支持反馈 [15]，针对单个模型输出，但忽略了多智能体场景中的传播动态。

**多智能体协作方法**，采用多数投票策略 [27] 或不确定性估计 [14]。然而，它们简化了智能体依赖关系，没有完全捕获多智能体系统中的复杂依赖关系。此外，它们需要修改基础模型，限制了在闭源模型上的适用性。

> [!note] 译者注（非原文）
> 论文此处强调了现有方法的核心缺陷：(1) 把每个智能体的输出独立看待，忽略信息传播的级联效应；(2) 需要访问模型内部，不适用于 GPT-4 等闭源 API 模型。GUARDIAN 正是为了同时解决这两个问题而设计的。

### GUARDIAN 的核心思路

本文旨在开发一种统一方法，用于检测和缓解 LLM 协作过程中的多种安全问题，包括幻觉放大和错误注入与传播。为实现这一目标，本文提出：

**时序属性图建模**：将多智能体协作过程建模为离散时序属性图 (discrete-time temporal attributed graph)，其中节点代表不同时间步的智能体，边代表智能体间通信，节点属性编码智能体响应。这种表示方式使得通过节点文本属性显式建模幻觉和错误的传播动态成为可能，同时使协作过程明确且透明。

**无监督编码器-解码器架构**：基于此图框架，提出一种基于编码器-解码器架构的无监督学习范式。该模型学习从潜在嵌入 (latent embeddings) 中重建节点属性和图结构，从而识别偏离正常模式的异常，以前所未有的精度精确定位潜在的幻觉或错误。

**图抽象机制**：基于信息瓶颈理论 (Information Bottleneck Theory) [28] 开发了一种图抽象机制。据作者所知，这是首次将 IB 原则系统性地应用于缓解基于 LLM 的多智能体协作中的安全风险。该机制通过过滤冗余或无关信息来压缩时序交互图，同时保留对异常检测最关键的必要模式。

> [!tip] 理解提示（非原文）
> 信息瓶颈理论的核心思想：在压缩输入信息的同时，尽可能保留与任务目标相关的信息。用公式表示即最小化 $I(X;Z) - \beta I(Z;Y)$，其中 $Z$ 是压缩表示，$Y$ 是目标变量。第一项鼓励压缩，第二项鼓励保留任务相关信息。

**理论分析**：进一步揭示了智能体间的信息流是有界的，历史状态与当前状态之间的转换受先验约束引导。

**增量训练范式**：采用与多智能体协作顺序性质一致的增量训练范式。在该范式中，模型逐步从交互历史中学习，同时动态适应随时间推移出现的新模式。通过在连续时间步间持续微调神经网络并从输入图中移除先前识别的异常，该方法使模型能够在动态演进的环境中积累知识并改进其异常检测能力。

> [!warning] 易混点（非原文）
> "增量训练"不同于持续学习 (continual learning) 中防止灾难性遗忘的技术。这里指的是：每新增一个时间步的交互数据就更新模型，且已检测到的异常节点从后续训练数据中移除——是一种"边检测、边学习、边清洗"的在线学习策略。

**模型无关性**：GUARDIAN 保持了模型无关的运行方式，无需修改底层 LLM，能够兼容不同的语言模型，无论其架构或访问级别如何。

### 实验效果概述

大量实验证明了所提出的 GUARDIAN 在保护 LLM 多智能体协作免受多种安全漏洞影响方面的有效性。该模型在保持高效资源利用（通过优化的 API 调用和计算开销）的同时，达到了最先进的准确率。时序图框架还使信息流动态的透明可视化成为可能，为多智能体交互提供了关键洞察。
