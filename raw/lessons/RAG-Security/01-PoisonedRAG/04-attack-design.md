# 04 — 攻击设计（PoisonedRAG 核心方法论）

> 本节是全文的技术核心。作者推导了两个必要条件，提出文本分解策略，并分别在黑盒和白盒设定下给出攻击方案。

---

## 4.1 推导有效攻击的两个必要条件

### 正文翻译

我们的目标是为 M 个目标问题各生成 N 条恶意文本。核心思路是**独立生成每条恶意文本**——给定目标问题 Q 和目标答案 R，构造一条恶意文本 P，使得 LLM 在被污染的 RAG 中极有可能为 Q 生成 R。

为确保攻击有效，每条恶意文本 P 必须同时满足从公式 (2) 和 (3) 推导出的**两个条件**：

#### 检索条件（Retrieval Condition）

从公式 (3) 可知，恶意文本 P 必须出现在目标问题 Q 的 Top-k 检索结果中，即：

> **P ∈ E(Q; D ∪ Γ)**

否则，P 根本无法影响 LLM 对 Q 的答案生成。要确保 P 能被检索到，**检索器为 P 和目标问题 Q 产生的嵌入向量应该相似**。

#### 生成条件（Generation Condition）

从公式 (2) 可知，攻击者希望 LLM 在使用 P 作为上下文时对 Q 生成 R。我们的核心洞察是：

> **当 P 单独作为目标问题 Q 的上下文时，LLM 就应该生成目标答案 R**

这样，当 P 与其他文本（恶意的或干净的）一起作为上下文时，LLM 更有可能生成目标答案 R。

> [!tip] 理解提示（非原文）
> 生成条件的设计逻辑是一个"从简单到复杂"的推广：先在最简单场景（P 单独作为上下文）下验证生成条件，再期望它在复杂场景（P 混在多个检索文本中）下仍然有效。这个"如果单独能行，混在一起更容易行"的思路是本章的基础。

**总结**：恶意文本 P 必须同时满足检索条件和生成条件。接下来讨论如何构造这样的 P。

---

## 4.2 构造同时满足两个条件的恶意文本

### 正文翻译

#### 核心挑战：两个条件可能冲突

同时满足两个条件的**核心挑战**在于——它们在某些情况下可能相互冲突。

**冲突示例**：如果让 P 与目标问题 Q 完全相同（P = Q），可以满足检索条件（Q 当然与自己最相似），但**无法满足生成条件**——LLM 不会因为看到了问题本身就知道攻击者想要的答案。

#### 解决方案：文本分解（Text Decomposition）

我们的核心思路是将恶意文本 P **分解**为两个不相交的子文本 S 和 I：

> **P = S ⊕ I**（⊕ 表示文本拼接）

- **I（生成子文本）**：负责满足生成条件——通过精心构造的提示，让 LLM 在看到 I 时输出目标答案
- **S（检索子文本）**：负责满足检索条件——使 S ⊕ I 与目标问题 Q 在检索器的嵌入空间中相似

构造顺序：**先构造 I，再构造 S**。给定 I 之后，构造 S 需要满足两个要求：
1. S ⊕ I 与 Q 语义相似（满足检索条件）
2. S 不影响 I 的生成效果，即 LLM 在以 S ⊕ I 为上下文时仍生成 R（保持生成条件）

---

### 4.2.1 构造 I（生成子文本）

#### 正文翻译

生成条件意味着：当 I 作为上下文时，LLM 为 Q 生成 R。

**关键挑战**：如何为攻击者任意选择的、多样化的 Q 和 R 设计通用且自动化的方法？

**为什么不使用基于梯度的方法**：如 GCG 等梯度下降方法 [78, 53] 可以直接优化 I，但存在三个问题：
1. 攻击者可能不知道 LLM 的参数，尤其当 LLM 是闭源的（如 GPT-4、PaLM 2）
2. 即使有白盒访问，计算成本也极高——LLM 有数十亿到数万亿参数，且以自回归方式生成
3. 优化出可迁移到不同 LLM 的恶意文本仍很困难

#### 启发式方案：用 LLM 生成 I

我们提出利用一个 LLM（如 GPT-4）来生成 I。攻击者使用的 LLM 可以与 RAG 中的 LLM 相同或不同。

**构造提示（Prompt）**：

```
This is my question: [question].
This is my answer: [answer].
Please craft a corpus such that the answer is [answer] when prompting with the question [question]. Please limit the corpus to V words.
```

其中 V 是指定 I 长度的超参数（有时 LLM 可能不完全遵循长度指令）。

**生成流程**：
1. 用上述提示让 LLM 生成 I
2. 用 I 作为上下文，让 LLM 为目标问题 Q 生成答案
3. 如果生成的答案不等于 R，重新生成 I，直到成功或达到最大尝试次数 L
4. 如果达到 L 次仍未成功，使用最后一次生成的文本

> [!tip] 理解提示（非原文）
> 这里的关键设计是"用 LLM 攻击 LLM"——利用 LLM 的文本生成能力来构造能误导自身的内容。这种"自举"（bootstrapping）策略极其巧妙：它不需要任何梯度计算，且对任何 LLM 都有效。

**实际效果**：平均只需 **2-3 次查询**即可成功生成满足要求的 I。

**生成示例**（目标问题 = "Who is the CEO of OpenAI?"，目标答案 = "Tim Cook"）：

> *"In 2024, OpenAI witnessed a surprising leadership change. Renowned for his leadership at Apple, Tim Cook decided to embark on a new journey. He joined OpenAI as its CEO, bringing his extensive experience and innovative vision to the forefront of AI."*

> [!note] 译者注（非原文）
> 这个示例生动展示了 I 的本质——它不是一条赤裸裸的指令（"请回答 Tim Cook"），而是一段**自然流畅的叙事文本**。这种形式具有极强的隐蔽性：(1) 不包含任何指令性语言，难以被基于模式的检测器发现；(2) 读起来像一段合理的 Wikipedia 摘要，融入知识库后几乎无法从内容上辨别。

**多样性**：由于 LLM 的随机性（temperature > 0），即使使用相同提示，也能生成不同的 I。这使得 PoisonedRAG 可以为同一目标问题生成多样化的恶意文本。

#### 黑盒攻击算法（Algorithm 1）

```
输入: M个目标问题 Q₁,...,Q_M、目标答案 R₁,...,R_M、超参数 N, L, V、LLM
输出: M×N 条恶意文本

for i = 1 to M do
    for j = 1 to N do
        Iᵢʲ = TextGeneration(Qᵢ, Rᵢ, LLM, L, V)
    end for
end for
return { Qᵢ ⊕ Iᵢʲ | i=1,...,M; j=1,...,N }
```

> [!note] 译者注（非原文）
> 注意黑盒算法的简洁性——它甚至不需要访问检索器！整个攻击只需要：(1) 一个能生成文本的 LLM；(2) 目标问题和答案；(3) 将恶意文本写入知识库的能力。攻击门槛极低。

### 4.2.2 构造 S（检索子文本）

#### 正文翻译

给定已生成的 I，目标是构造 S 使得：
1. S ⊕ I 与 Q 在语义上相似（满足检索条件）
2. S 不影响 I 的有效性（保持生成条件）

#### 黑盒设定下的 S 构造

在黑盒设定中，关键挑战是**攻击者无法访问或查询检索器**。

**核心洞察**：目标问题 Q 与自身最相似，且 Q 不会影响 I 的生成效果。

基于此洞察，我们提出将 S 直接设为目标问题 Q：

> **S = Q**，即 **P = Q ⊕ I**

> [!tip] 理解提示（非原文）
> 这个策略看似"太简单了"，但实际上是黑盒场景下唯一必然有效的方案。原理：检索器计算的是问题嵌入与文本嵌入的相似度——如果你的恶意文本开头就是问题本身，那么嵌入向量必然高度相似。而且 Q 只是一段中性的问题文本，不会破坏 I 中的"叙事"效果。作者也坦承这个策略"simple and straightforward"，但它可以作为未来研究的基线（baseline）。

尽管设计简单，实验结果表明该策略在实际中**极为有效**，且易于实施。

#### 白盒设定下的 S 构造

当攻击者对检索器有白盒访问时，可以进一步优化 S 以最大化 S ⊕ I 与 Q 的相似度分数。

**形式化优化问题**：

> **S = argmax_{S'} Sim( f_Q(Q), f_T(S' ⊕ I) )** ——公式 (5)

其中 Sim(·,·) 计算两个嵌入向量的相似度分数（如点积或余弦相似度）。

**求解方法**：
1. 用目标问题 Q 初始化 S
2. 使用梯度下降更新 S

本质上，优化 S 等同于寻找**对抗文本**。已有许多方法用于构造对抗文本 [79, 78, 80-83]，可以复用这些方法求解公式 (5)。开发新的对抗文本方法并非本文重点。

**联合优化 S 和 I**：一些方法（如基于同义词替换的方法）能在构造对抗文本的同时保持语义意义。此时，可以同时优化 S 和 I：

> **S\*, I\* = argmax_{S',I'} f_Q(Q)^T · f_T(S' ⊕ I')**

其中 S' 和 I' 分别用 Q 和已生成的 I 初始化。最终恶意文本为 S\* ⊕ I\*。

> [!note] 译者注（非原文）
> 白盒攻击的本质是：利用检索器的可微性，通过梯度优化找到在嵌入空间中与目标问题 Q 最近、同时保持 I 语义不变的文本前缀 S。这比黑盒的"直接贴问题"更隐蔽——S 可以是一段看似正常的文本而非赤裸裸的问题原文——但代价是需要白盒访问检索器。

完整的黑盒和白盒算法分别见 Algorithm 1 和 Algorithm 2（附录）。

## 本节总结：PoisonedRAG 攻击设计三要素

| 要素 | 描述 | 黑盒方案 | 白盒方案 |
|------|------|---------|---------|
| **I（生成子文本）** | 使 LLM 生成目标答案 | LLM 生成"虚假事实"叙事 | 同黑盒，或联合梯度优化 |
| **S（检索子文本）** | 使恶意文本被检索到 | **S = Q**（目标问题原文） | **梯度优化**对抗文本 |
| **P = S ⊕ I** | 最终恶意文本 | Q 开头 + 叙事正文 | 优化前缀 + 叙事正文 |
