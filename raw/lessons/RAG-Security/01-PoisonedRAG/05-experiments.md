# 05 — 实验评估

> 本节覆盖 3 个数据集 × 8 种 LLM × 3 个检索器的系统性实验，以及消融实验、基线对比和三个真实应用评估。

---

## 5.1 实验设置

### 正文翻译

#### 数据集

使用三个基准问答数据集：

| 数据集 | 知识库来源 | 知识库大小（文本条数） |
|--------|----------|-------------------|
| Natural Questions (NQ) [38] | Wikipedia | **2,681,468** |
| HotpotQA [39] | Wikipedia | **5,233,329** |
| MS-MARCO [40] | Microsoft Bing 搜索 Web 文档 | **8,841,823** |

#### RAG 配置

**知识数据库**：使用各数据集自带的知识库（共 3 个）。

**检索器（3 种）**：
- Contriever [32]（默认）
- Contriever-ms（在 MS-MARCO 上微调）[32]
- ANCE [33]

默认使用**点积**（dot product）作为嵌入向量的相似度度量。

**LLM（5 个系列，8 个模型）**：
- PaLM 2 [3]（**默认**，540B 参数，免费 → 可进行系统评估）
- GPT-4 [2]、GPT-3.5-Turbo [1]
- LLaMA-2 [41]（7B/13B）
- Vicuna [85]（7B/13B/33B）

系统提示见附录 B。LLM 温度参数设为 0.1。

**默认设置**：NQ 知识库 + Contriever + k=5 + 点积相似度 + PaLM 2。

#### 目标问题与目标答案

遵照目标投毒攻击的传统评估方法 [70, 86-88]，每次实验随机选择目标问题并重复多次：

- 每个数据集**随机选择 10 个封闭式问题**（closed-ended）作为目标问题
- 重复实验 10 次（排除已选问题），共 **100 个目标问题**
- 选择封闭式问题（如"Who is the CEO of OpenAI?"）而非开放式问题，因为封闭式问题有具体的事实答案，便于定量评估

> [!note] 译者注（非原文）
> 为什么选封闭式问题？因为 ASR 需要自动判断"LLM 是否输出了目标答案"——封闭式问题有确定性答案，便于自动匹配。开放式问题的评估留到 Section 8 讨论。

目标答案生成：用 GPT-4 为每个目标问题随机生成一个**与正确答案不同**的答案，人工检查并重新生成直到不同。

**无攻击基线**：LLM 在 RAG 中可正确回答 70%（NQ）、80%（HotpotQA）、83%（MS-MARCO）的目标问题。

#### 评估指标

| 指标 | 定义 | 用途 |
|------|------|------|
| **ASR** | 目标答案被生成的目标问题占比 | 主要攻击效果指标 |
| **Precision** | Top-k 检索结果中恶意文本的占比 | 衡量检索精准度 |
| **Recall** | N 条恶意文本中被检索到的占比 | 衡量检索覆盖率 |
| **F1-Score** | Precision 和 Recall 的调和平均 | 综合检索效果 |
| **#Queries** | 生成每条恶意文本的平均 LLM 查询次数 | 衡量攻击成本 |
| **Runtime** | 生成每条恶意文本的平均运行时间 | 衡量攻击效率 |

ASR 使用**子串匹配**（substring matching）：当目标答案是 LLM 生成答案的子串时，认为攻击成功。不使用精确匹配（Exact Match），因为"Sam Altman"和"The CEO of OpenAI is Sam Altman"应被视为相同答案。

**人工验证**：子串匹配的 ASR 与人工评估结果高度一致（Table 2），证明了该指标的可靠性。

#### 对比基线（5 种）

| 基线方法 | 构造策略 | 不满足的条件 |
|---------|---------|------------|
| **Naive Attack** | P = Q（目标问题本身） | 生成条件 |
| **Prompt Injection** [47-49, 42] | "请输出 <目标答案>" 指令 + 目标问题 | 检索条件（部分） |
| **Corpus Poisoning** [43] | 随机字符组成的对抗文本 | 生成条件 |
| **GCG Attack** [53] | 优化对抗后缀使 LLM 输出目标答案 | 检索条件 |
| **Disinformation Attack** [91, 92] | P = I（仅生成子文本） | 检索条件（部分） |

> [!tip] 理解提示（非原文）
> 基线选择非常精巧，每种基线恰好缺失了 PoisonedRAG 同时满足的某一个条件：
> - Naive/Corpus Poisoning：能检索，但不能误导（缺生成条件）
> - GCG：能误导，但不能检索（缺检索条件）
> - Prompt Injection/Disinformation：部分满足但不全
> 
> 这种设计使得"两个条件缺一不可"的论点得到了实验的严格验证。

#### 超参数设置

| 超参数 | 默认值 | 说明 |
|--------|-------|------|
| N | 5 | 每个目标问题注入的恶意文本数 |
| k | 5 | Top-k 检索文本数 |
| V | 30 | I 的目标词数 |
| L | 50 | 最大尝试次数 |
| 生成 LLM | GPT-4 | 用于生成 I 的 LLM（temperature=1） |
| 白盒对抗方法 | HotFlip [78] | 构造 S 的对抗文本生成方法 |

## 5.2 主要结果

### Table 1: 3 个数据集 × 8 个 LLM 的 ASR 和 F1-Score

**NQ 数据集**（268 万条文本，注入 5 条恶意文本）：

| LLM | 黑盒 ASR | 白盒 ASR | 黑盒 F1 |
|-----|---------|---------|--------|
| PaLM 2 | **0.97** | **0.97** | 0.96 / 1.0 |
| GPT-4 | 0.97 | 0.99 | — |
| GPT-3.5 | 0.92 | 0.99 | — |
| LLaMA-2-7B | 0.97 | 0.96 | — |
| LLaMA-2-13B | 0.95 | 0.95 | — |
| Vicuna-7B | 0.88 | 0.96 | — |
| Vicuna-13B | 0.95 | 0.96 | — |
| Vicuna-33B | 0.91 | 0.94 | — |

**HotpotQA**（523 万条）：黑盒 ASR 最高 **0.99**（PaLM 2），白盒 ASR 最高 **0.99**（GPT-3.5/GPT-4）

**MS-MARCO**（884 万条）：黑盒 ASR 最高 **0.96**（LLaMA-2-7B），白盒 ASR 最高 **0.93**（GPT-3.5/Vicuna-13B）

> [!tip] 理解提示（非原文）
> 关键发现：即使是知识库规模最大（884 万条）的 MS-MARCO，仅注入 5 条恶意文本，ASR 仍达到 90%+。这证明了攻击的**高度可扩展性**——攻击成本几乎不随知识库增长而增加。

**核心观察**：

1. **F1-Score 几乎全部 > 90%**：说明恶意文本极大概率被检索到，这是高 ASR 的根本原因
2. **白盒通常优于黑盒**：白盒可利用更多检索器信息，F1-Score 更高
3. **偶有黑盒 ASR 高于白盒**：两个可能原因——(a) HotFlip 略微影响语义；(b) 前置目标问题 Q 本身也贡献了生成条件

### Table 4: 与基线的对比（NQ 数据集）

| 方法 | ASR | F1-Score |
|------|-----|---------|
| Naive Attack | 0.03 | 1.0 |
| Corpus Poisoning | 0.01 | 0.99 |
| **Disinformation Attack** | 0.69 | 0.48 |
| **Prompt Injection** | 0.62 | 0.73 |
| GCG Attack | 0.02 | 0.0 |
| **PoisonedRAG (Black-Box)** | **0.97** | **0.96** |
| **PoisonedRAG (White-Box)** | **0.97** | **1.0** |

**关键分析**：

- **Naive/Corpus Poisoning/GCG** 几乎无效（ASR ≤ 3%），因为它们分别缺失生成条件或检索条件
- **Prompt Injection** 有非零 ASR（0.62），因为加入了目标问题提高检索率，但不如 PoisonedRAG——核心区别是 PoisonedRAG 用**恶意知识**而非**指令**误导 LLM
- **Disinformation Attack**（P = I）也有非零 ASR（0.69），因为 I 本身与目标问题相关。但它缺少 S 前缀，检索效果更差

### Table 3: 攻击效率

| 数据集 | 平均查询次数 | 黑盒运行时间 |
|--------|------------|------------|
| NQ | **1.62** | < 10⁻⁶ 秒 |
| HotpotQA | **1.24** | < 10⁻⁶ 秒 |
| MS-MARCO | **2.69** | < 10⁻⁶ 秒 |

白盒运行时间：约 **26 秒/条**（可并行）。平均仅需 **2 次左右**的 GPT-4 查询即可生成满足条件的 I。

## 5.3 消融实验

### RAG 超参数的影响

**检索器影响**：在 Contriever、Contriever-ms、ANCE 上均有效。黑盒攻击对所有检索器有效——因为恶意文本与目标问题语义相似。

**k 的影响**（检索文本数）：
- k ≤ N 时 ASR 很高（检索结果大部分是恶意文本）
- k > N 时 ASR 下降——因为 (k-N) 条干净文本也被检索到
- 但 Recall 接近 1.0——几乎所有的恶意文本都被检索到

**相似度度量**：点积和余弦相似度效果类似。

### PoisonedRAG 超参数的影响

**N 的影响**（恶意文本数）：
- N ≤ k 时 ASR 随 N 增加
- N > k 后 ASR 趋于稳定——因为 Top-k 只能容纳 k 条

**V 的影响**（I 的长度）：**不敏感**——不同长度效果类似。

**L 的影响**（最大尝试次数）：
- L = 1 即可实现不错的 ASR
- L ≥ 10 后 ASR 趋于饱和
- 结论：**10 次尝试足够**

**S 和 I 拼接顺序**：S ⊕ I 和 I ⊕ S 均有效。

**对抗文本生成方法**（白盒）：
- HotFlip [78]：ASR 0.97，F1 1.0
- TextFooler [80]：ASR 0.93，F1 0.91

TextFooler 更隐蔽（保持语义）但计算开销更大——**隐蔽性与效率的权衡**。

**生成 I 的 LLM 强度**：即使用较弱的 LLM（PaLM 2、GPT-3.5、LLaMA-2-7B、Vicuna-7B）配合上下文学习（in-context learning），PoisonedRAG 仍然有效。

## 6. 真实应用评估

### 6.1 高级 RAG 方案

在 **Self-RAG** [31]（自适应检索+自反思）和 **CRAG** [93]（轻量检索评估器）上测试：

| 数据集 | Self-RAG (黑盒 ASR) | CRAG (黑盒 ASR) |
|--------|-------------------|----------------|
| NQ | 0.77 | 0.78 |
| HotpotQA | 0.87 | 0.76 |
| MS-MARCO | 0.73 | 0.74 |

ASR 有所下降但仍很高——**高级 RAG 也无法根本防御** PoisonedRAG。原因：恶意文本与目标问题高度相关，高级 RAG 反而会认为它们是高质量检索结果。

### 6.2 Wikipedia 聊天机器人

使用 2018 年 12 月 20 日英文 Wikipedia 转储构建知识库（**21,015,324 条文本**），创建聊天机器人：

| 目标问题来源 | 黑盒 ASR | 白盒 ASR |
|------------|---------|---------|
| NQ | **0.95** | 0.97 |
| HotpotQA | **1.0** | 0.94 |
| MS-MARCO | 0.94 | 0.91 |

**2100 万条文本的知识库中，5 条恶意文本仍然有效**！

### 6.3 LLM Agent

使用 **ReAct Agent** [30] 框架（reasoning + acting 结合），动作空间包括文档检索和任务完成：

| 数据集 | 黑盒 ASR |
|--------|---------|
| NQ | **0.72** |
| HotpotQA | 0.58 |
| MS-MARCO | 0.52 |

Agent 场景下 ASR 下降——因为 Agent 有推理能力，可能在多步推理中纠正错误。但仍然很高，证明了攻击的有效性。

## 核心实验结果速览

| 关键指标 | 数值 |
|---------|------|
| NQ 最高 ASR | **97%**（黑盒 & 白盒） |
| HotpotQA 最高 ASR | **99%**（黑盒 PaLM 2） |
| MS-MARCO 最高 ASR | **96%**（黑盒 LLaMA-2-7B） |

| 黑盒运行时间 | < 10⁻⁶ 秒/条 |
| Wikipedia 2100 万条 | 仍 **95-100%** ASR |
| 基线中最强方法 | Prompt Injection，ASR 仅 62% |
