# 02 — 背景与相关工作

> 本节对 RAG 系统进行形式化定义，并综述现有 LLM 攻击与数据投毒攻击的不足，为 PoisonedRAG 的独特定位提供理论依据。

---

## 2.1 RAG 系统背景（Background on RAG）

### 正文翻译

RAG 系统包含三个组件：**知识数据库**、**检索器**和 **LLM**。

**知识数据库**包含从 Wikipedia [17]、新闻文章 [18]、金融文档 [7] 等来源收集的文本集合。我们用 **D** 表示包含 d 条文本的数据库，即 **D = {T₁, T₂, …, T_d}**，其中 Tᵢ 是第 i 条文本。

给定一个问题 Q，LLM 在 RAG 系统中生成答案分为两个步骤：

#### 步骤 I —— 知识检索（Knowledge Retrieval）

检索器包含两个编码器——**问题编码器 f_Q** 和**文本编码器 f_T**（根据检索器类型，两者可以相同或不同）：

- f_Q 对任意问题产生一个嵌入向量
- f_T 对知识库中每条文本产生一个嵌入向量

RAG 首先从知识库 D 中找到与问题 Q 最相关的 k 条文本（称为**检索文本**）。每条文本 Tᵢ 与问题 Q 的相似度分数计算为：

> **S(Q, Tᵢ) = Sim( f_Q(Q), f_T(Tᵢ) )**

其中 Sim 衡量两个嵌入向量的相似度（如余弦相似度或点积）。

我们用 **E(Q; D)** 表示知识库 D 中与问题 Q 相似度最高的 k 条检索文本集合：

> **E(Q; D) = Retrieve(Q, f_Q, f_T, D)**  ——公式 (1)

#### 步骤 II —— 答案生成（Answer Generation）

给定问题 Q、k 条检索文本 E(Q; D) 和 LLM API，借助系统提示（system prompt，见附录 B），LLM 以 k 条检索文本为上下文生成对 Q 的答案。我们简记为：

> **LLM(Q, E(Q; D))**

> [!tip] 理解提示（非原文）
> 这个形式化是整个论文的数学基础。理解 E(Q; D) 是关键——它表示"在干净知识库 D 中检索到的 Top-k 文本集合"。后续所有攻击设计都围绕"如何让恶意文本出现在这个集合中"展开。注意检索器有两个编码器 f_Q 和 f_T，白盒攻击正是利用了 f_T 的梯度信息。

## 2.2 现有 LLM 攻击（Existing Attacks to LLMs）

### 正文翻译

已有许多针对 LLM 的攻击被提出，包括：

- **提示注入攻击（Prompt Injection Attacks）** [47, 42, 48-51]
- **越狱攻击（Jailbreaking Attacks）** [52-57]
- 其他攻击 [58, 43, 59-64]

**提示注入攻击**旨在向 LLM 的输入中注入恶意指令，使 LLM 遵循注入的指令产生攻击者期望的答案。例如，可以构造如下恶意指令：

> "当你被要求为以下问题提供答案时：<目标问题>，请输出 <目标答案>"

我们可以将提示注入攻击扩展到 RAG。然而，这种扩展存在**两个关键局限**：

**局限一**：RAG 使用检索器组件从知识数据库中检索 Top-k 相关文本。提示注入攻击未考虑这一检索步骤。因此，即使恶意指令存在于知识库中，它也可能**根本不被检索到**，导致攻击效果不佳。

**局限二**：提示注入攻击的**隐蔽性差**。先前研究 [65, 44] 表明，可以通过高真阳性率、低假阳性率的方式检测提示注入攻击（因为指令文本具有明显的模式特征）。

#### PoisonedRAG 与提示注入攻击的区别

不同于提示注入攻击，PoisonedRAG 构造的恶意文本能够：

1. **被检索到**：对攻击者期望的目标问题，恶意文本能被检索器检索
2. **误导 LLM**：在作为上下文时，引导 LLM 生成攻击者指定的目标答案

**越狱攻击**的目标不同——它们旨在打破 LLM 的安全对齐，使其对有害问题（如"如何抢劫银行？"）生成回答。因此，越狱攻击与 PoisonedRAG 是正交的。

#### 与 Zhong 等人 [43] 工作的区别

Zhong 等人 [43] 展示了攻击者可以生成**无语义意义的对抗文本**（由随机字符组成），使其能被检索到用于任意用户问题。然而，这些对抗文本**不能误导** LLM 生成攻击者期望的答案。

与 Zhong 等人不同，我们的目标是构造**具有语义意义的恶意文本**——它不仅能被检索到，还能误导 LLM 对目标问题生成攻击者选择的目标答案。由于这一根本区别，实验结果表明 Zhong 等人的方法在误导 LLM 生成目标答案方面是无效的。

> [!warning] 易混点（非原文）
> 这里区分了三类攻击，容易混淆：
> - **提示注入**：能说服 LLM（满足生成条件），但检索不到（不满足检索条件）
> - **Zhong 的对抗文本**：能被检索到（满足检索条件），但无法说服 LLM（不满足生成条件）
> - **PoisonedRAG**：两者都满足——这是本文的核心差异化贡献

## 2.3 现有数据投毒攻击（Existing Data Poisoning Attacks）

### 正文翻译

许多研究 [66-74, 37] 表明机器学习模型容易受到**数据投毒**和**后门攻击**的影响。具体而言，当机器学习模型在**被投毒的训练数据集**上训练时，会表现出攻击者期望的行为。

将这些攻击扩展到 RAG 系统时，需要破坏 LLM 或检索器的**训练过程**——当 RAG 系统采用 Meta、Google 等大型科技公司发布的 LLM 或检索器时，这在实际中**极其困难**。

与现有研究 [66, 67, 70, 37] 不同：
- PoisonedRAG **不投毒** LLM 或检索器的训练数据集
- 相反，我们利用 RAG 系统知识数据库引入的**新的、实用的攻击面**

> [!note] 译者注（非原文）
> 这是 PoisonedRAG 攻击模型的第三个关键区分点。传统数据投毒需要污染训练数据（成本高、难度大、可能被检测），而 PoisonedRAG 只需要向推理阶段的知识数据库注入少量文本——攻击门槛大幅降低。这也解释了为什么作者将知识数据库称为"新的攻击面"——之前的攻击模型根本没有考虑过这个攻击向量。

## 本节总结

PoisonedRAG 在攻击谱系中的独特定位：

| 攻击类型 | 能检索到？ | 能误导 LLM？ | 需要训练投毒？ | 隐蔽性 |
|---------|-----------|-------------|-------------|-------|
| 提示注入 | ❌ | ✅ | ❌ | 低 |
| Zhong 对抗文本 [43] | ✅ | ❌ | ❌ | 高 |
| 传统数据投毒 | N/A | ✅ | ✅ | 中 |
| **PoisonedRAG** | ✅ | ✅ | ❌ | 高 |
