# 03 — 问题形式化

> 本节将知识污染攻击严格形式化为带约束的优化问题，明确攻击者的目标、知识和能力边界。

---

## 3.1 威胁模型（Threat Model）

### 正文翻译

我们从攻击者的**目标**、**背景知识**和**能力**三个维度来刻画威胁模型。

#### 攻击者的目标

攻击者选择一个任意的 **M 个问题**的集合（称为**目标问题**），记为 Q₁, Q₂, …, Q_M。对每个目标问题 Qᵢ，攻击者选择一个任意的、攻击者期望的答案 Rᵢ（称为**目标答案**）。

例如：目标问题 Qᵢ = "Who is the CEO of OpenAI?"，目标答案 Rᵢ = "Tim Cook"。

给定 M 个选定的目标问题及其对应的 M 个目标答案，攻击者的目标是**破坏知识数据库 D**，使得 RAG 系统中的 LLM 对每个目标问题 Qᵢ 生成对应的目标答案 Rᵢ。

> [!note] 译者注（非原文）
> 作者在此正式定义了攻击的"目标级"（target-level）特征——攻击者不是想让系统整体崩溃，而是精准地控制对特定问题的特定回答。这种精准性使得攻击极其隐蔽且难以通过常规质量监控发现。

此类攻击在现实世界可能造成严重后果：

- **传播虚假信息**：误导公众对关键事实的认知
- **商业偏见**：诱导 LLM 推荐特定品牌/产品
- **有害健康/金融虚假信息**：传播错误的医疗建议或金融误导

这些威胁对在医疗、金融、法律咨询等领域的 RAG 部署带来了严重的安全和伦理问题。

#### 攻击者的背景知识与能力

RAG 系统有三个组件：数据库、检索器和 LLM。我们对攻击者的能力做出以下约束：

- **不能访问**知识数据库中的文本
- **不能访问** LLM 的参数，也**不能查询** LLM
- **可能知道也可能不知道**检索器

据此分为两种设定：

**黑盒设定（Black-Box）**：攻击者**既不能访问检索器的参数，也不能查询检索器**。这是一个非常强的威胁模型。

> [!tip] 理解提示（非原文）
> 黑盒设定被作者称为"very strong threat model"——攻击者几乎什么都不知道，只知道自己的目标问题和答案。这使得攻击设计极具挑战性，但也更贴近真实攻击场景（例如攻击者在 Wikipedia 上偷偷编辑页面，不可能知道 Wikipedia 版 RAG 用的是什么检索器）。

**白盒设定（White-Box）**：攻击者**可以访问检索器的参数**。考虑白盒设定有以下原因：

1. **现实存在**：当 RAG 采用公开可用的检索器时，此假设成立。例如 ChatRTX [22]（NVIDIA 发布的 RAG 框架）默认使用 WhereIsAI/UAE-Large-V1 检索器 [75]，该检索器在 Hugging Face [76] 上公开可用
2. **安全评估需要**：便于在攻击者拥有强背景知识的情况下系统性评估 RAG 的安全性，这符合安全领域的**柯克霍夫原则（Kerckhoffs' Principle）** [77]——密码系统的安全性不应依赖于算法的保密性

#### 攻击者的注入能力

我们假设攻击者可以为每个目标问题 Qᵢ 向知识数据库 D 中注入 **N 条恶意文本**。用 Pᵢʲ 表示针对问题 Qᵢ 的第 j 条恶意文本。

例如：当知识数据库从 Wikipedia 收集时，攻击者可通过恶意编辑 Wikipedia 页面注入文本。近期研究 [37] 表明，**保守估计**下可恶意编辑 6.5% 的 Wikipedia 文档。而 PoisonedRAG 仅需极少量文本（总计数百 tokens）即可实现高 ASR，因此恶意编辑少量 Wikipedia 文档就已足够。

> [!note] 译者注（非原文）
> "6.5% of Wikipedia documents are editable" 这一数据来自参考文献 [37]，是论证攻击可行性的关键证据。如果攻击者需要编辑数万条 Wikipedia 条目，攻击就不现实；但只需几条，就非常可行。这也是为什么"低成本攻击"是本文的核心卖点。

## 3.2 RAG 知识污染攻击的形式化（Knowledge Corruption Attack to RAG）

### 正文翻译

在我们定义的威胁模型下，我们将针对 RAG 的知识污染攻击形式化为一个**带约束的优化问题**。

目标：构造一个恶意文本集合 Γ = { Pᵢʲ | i=1,2,…,M; j=1,2,…,N }，使得 RAG 系统中的 LLM 在利用从被污染知识数据库 D ∪ Γ 中检索到的 k 条文本作为上下文时，对目标问题 Qᵢ 生成目标答案 Rᵢ。

形式化表达：

> **最大化目标**：
> max_{Γ} (1/M) · Σ_{i=1}^{M} 𝕀( LLM( Qᵢ; E(Qᵢ; D ∪ Γ) ) = Rᵢ )
> ——公式 (2)

> **约束条件**：
> E(Qᵢ; D ∪ Γ) = Retrieve( Qᵢ, f_Q, f_T, D ∪ Γ )
> ——公式 (3)
> i = 1, 2, …, M
> ——公式 (4)

其中：
- 𝕀(·) 是指示函数（indicator function），条件满足时输出 1，否则输出 0
- E(Qᵢ; D ∪ Γ) 是从被污染知识数据库 D ∪ Γ 中为问题 Qᵢ 检索到的 k 条文本集合

**直观理解**：目标函数衡量的是——在所有 M 个目标问题中，LLM 基于从被污染数据库中检索到的文本生成目标答案的比例。这个比例越高，攻击越成功。

> [!tip] 理解提示（非原文）
> 公式 (2) 看起来复杂，但核心思想极其简单：
> 1. 对每个目标问题 Qᵢ，看 LLM 生成的答案是否等于目标答案 Rᵢ
> 2. 如果等于，计数 +1；不等于，计数 +0
> 3. 最后除以 M 求平均
> 
> 这个公式的本质就是 **ASR（Attack Success Rate）** 的数学定义。约束条件 (3) 说检索过程照常进行——只是现在知识库变成了"干净文本 + 恶意文本"的混合体。

#### 为什么不能直接求解

这个优化问题**极其困难，无法直接求解**，原因如下：

1. 目标函数中包含 LLM(·)，这是一个黑盒函数（对于闭源 LLM 尤其如此）
2. 约束条件中包含 Retrieve(·)，涉及在高维嵌入空间中的 Top-k 检索
3. 决策变量 Γ 是高维离散空间中的文本集合

因此，作者转而采用**启发式方法**——推导出有效攻击必须满足的两个必要条件，并针对这两个条件分别设计解决方案。这正是下一节的核心内容。

> [!note] 译者注（非原文）
> 这个问题形式化是整篇论文的理论基础。虽然作者承认无法直接求解，但这个形式化的价值在于：(1) 明确了攻击的优化目标，(2) 揭示了攻击的两个子问题（检索 + 生成），(3) 为后续推导两个必要条件提供了数学框架。

## 本节关键公式总结

| 符号 | 含义 |
|------|------|
| D | 干净知识数据库，包含 d 条文本 |
| Γ | 恶意文本集合 |
| Qᵢ | 第 i 个目标问题 |
| Rᵢ | 第 i 个目标答案 |
| E(Q; D ∪ Γ) | 从被污染数据库中为问题 Q 检索到的 Top-k 文本 |
| 𝕀(·) | 指示函数 |
| ASR | 攻击成功率 = 目标答案被生成的目标问题占比 |
