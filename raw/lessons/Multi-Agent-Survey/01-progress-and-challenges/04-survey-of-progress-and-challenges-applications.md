# LLM 多智能体综述中文精读（四）：应用

> 本篇覆盖原论文 **Section 4 Applications**。作者把现有应用归纳成两条主线：**面向问题求解（Problem Solving）** 与 **面向世界模拟（World Simulation）**。

## 本章导读

> [!note] 译者注（非原文）
> 这一章最值得你建立的不是“论文名清单”，而是一个判断框架：
> - 有些 multi-agent 系统的目标是**把任务做成**；
> - 有些 multi-agent 系统的目标是**把复杂世界演出来**。
>
> 这两类系统在环境设计、角色设定、通信方式、评测指标上都可能很不一样。

## 正文翻译

> [!warning] 易混点（非原文）
> 下文是原论文第 4 节的完整中文翻译；中间的解释都明确标为非原文。

### 原文标题：4 Applications

LLM 多智能体系统已经被用于广泛的应用场景。作者在表 1 中总结了两类应用：**问题求解（Problem Solving）** 与 **世界模拟（World Simulation）**。下面作者对这些应用进行详细说明。

需要注意的是，这是一个增长极快的研究领域，新的应用几乎每天都在出现。作者维护了一个开源仓库来持续汇报最新工作。

### 原文标题：4.1 LLM-MA for Problem Solving

使用 LLM 多智能体进行问题求解的主要动机，是利用多个具有专门化专长的智能体的集体能力。这些智能体像不同个体一样协作，从而更有效地解决复杂问题，例如：

- 软件开发
- 具身智能体（embodied agents）
- 科学实验
- 科学辩论

下面作者分别介绍这些应用示例。

#### 原文标题：4.1.1 Software Development

软件开发本身就是一项复杂活动，需要产品经理、程序员、测试人员等多种角色协作完成。因此，LLM 多智能体系统通常会模拟这些不同角色，并通过协作来应对软件开发这一复杂挑战。

依照软件开发中的瀑布流（waterfall）或标准操作流程（Standardized Operating Procedures, SOPs）工作流，这类系统中的通信结构通常是**分层式（layered）**的。智能体通常会与代码解释器、其他智能体或人类交互，对生成代码进行迭代式优化。

作者依次提到几类代表性工作：

- [Li et al., 2023b] 最早提出一种简单的角色扮演式智能体框架，利用两个角色之间的互动，仅基于用户一句话指令实现自动化编程。这项工作为“可通信智能体的认知过程”提供了启发。
- [Dong et al., 2023b] 让 LLM 作为软件开发中不同子任务的“专家”协同工作，从而自主合作生成代码。
- [Qian et al., 2023] 提出一个端到端软件开发框架，使用多个智能体完成软件开发，但尚未引入高级的人类团队协作经验。
- [Hong et al., 2023] 首次将人类工作流经验显式引入多智能体编程系统，通过把 SOP 编码进提示词，来增强结构化协作与可控性。
- [Huang et al., 2023a] 进一步深入多智能体编程场景，重点处理“代码片段生成”与“测试用例生成、执行和优化”之间的平衡问题。

> [!note] 译者注（非原文）
> 软件开发会成为 multi-agent 的热门应用，并不奇怪，因为它天然适合“分工协作”叙事：
> - PM 负责需求理解；
> - Engineer 负责实现；
> - Tester 负责验证；
> - Reviewer 负责审查。
>
> 这类场景非常容易把人类团队结构映射到 agent 结构上，所以也是今天很多 agent framework 最爱展示的 demo 方向。

#### 原文标题：4.1.2 Embodied Agents

大多数具身智能体应用，本来就天然涉及多个机器人一起协作完成复杂的现实规划与操作任务，例如在异构机器人能力存在差异的仓储管理场景中。因此，LLM 多智能体可以用来建模具有不同能力的机器人，让它们彼此协作解决现实物理任务。

- [Dasgupta et al., 2023] 最早探索了使用 LLM 作为具身智能体行动规划器（action planner）的可能性。
- [Mandi et al., 2023] 提出 RoCo，这是一种新的多机器人协作方法。它利用 LLM 做高层通信，同时结合低层路径规划。每个机械臂都配有一个 LLM，并配合逆运动学与碰撞检测协作完成任务。实验结果表明，RoCo 在协作任务中具有良好的适应性与成功率。
- [Zhang et al., 2023c] 提出 CoELA（Cooperative Embodied Language Agent），它在 LLM 多智能体设定中管理讨论与任务规划。这一设定具有去中心化控制、复杂部分可观测性（partial observation）、高通信成本以及多目标长时程任务等难点。
- [Chen et al., 2023d] 研究了大量机器人参与时的通信难题，因为给每个机器人都分配一个 LLM 在成本与长上下文上都不现实。该研究比较了四种通信框架：中心化、去中心化以及两种混合模型，以评估它们在复杂多智能体任务中的协调效果。
- [Yu et al., 2023] 提出 Co-NavGPT，用于多机器人协同视觉目标导航，把 LLM 作为全局规划器，为每个机器人分配前沿目标。
- [Chen et al., 2023b] 提出一种基于 LLM 的寻求共识框架，可被用作多机器人聚合任务中的协作式规划器。

#### 原文标题：4.1.3 Science Experiments

就像多个智能体能够以不同专家身份合作解决软件开发和具身智能问题一样，多个智能体也可以组成一个“科学团队（science team）”来执行科学实验。

与前面应用相比，这一类应用的重要区别在于：**人类监督（human oversight）** 在其中至关重要。原因在于科学实验通常代价高昂，同时 LLM 智能体也存在幻觉（hallucination）问题。人类专家位于这些智能体系统的中心位置，负责处理来自智能体的信息，并向其提供反馈。

[Zheng et al., 2023] 使用多个基于 LLM 的智能体，每个智能体分别聚焦于科学实验中的不同任务，包括：

- 策略规划
- 文献检索
- 编码
- 机器人操作
- 实验器材设计

所有这些智能体都与人类互动，并协同优化复杂材料的合成过程。

> [!warning] 易混点（非原文）
> 在科学实验场景里，多智能体不只是“帮你多想几个答案”，而是会碰到**真实成本、实验风险与专家审查**。所以 human-in-the-loop 在这里不是锦上添花，而是系统安全边界的一部分。

#### 原文标题：4.1.4 Science Debate

LLM 多智能体还可以被设置到科学辩论场景中，让多个智能体通过相互辩论来提升集体推理能力。典型任务包括：

- MMLU
- 数学问题
- StrategyQA

其核心思想是：每个智能体先对问题给出自己的分析，然后进入联合辩论过程。经过多轮辩论之后，智能体会收敛到一个共识答案。

- [Du et al., 2023] 在六种不同的推理与事实性任务上利用多智能体辩论流程，结果表明这种辩论过程可以提升事实性。
- [Xiong et al., 2023] 聚焦常识推理任务，并构造了三阶段辩论流程，使其更贴近现实场景：公平辩论（fair debate）、不匹配辩论（mismatched debate）以及圆桌辩论（roundtable debate）。论文同时分析了不同 LLM 之间的一致性（inter-consistency），并指出辩论能够改善这种一致性。
- [Tang et al., 2023] 同样把多个基于 LLM 的智能体当作不同领域专家，让它们围绕一份医疗报告展开协同讨论，以达成医学诊断共识。

### 原文标题：4.2 LLM-MA for World Simulation

LLM 多智能体的另一类主流应用场景是**世界模拟（world simulation）**。这一方向的研究正在迅速增长，并横跨社会科学、游戏、心理学、经济学、政策制定等多个领域。

在世界模拟中使用 LLM 多智能体的关键原因，在于它们出色的**角色扮演能力（role-playing abilities）**。这种能力对于真实地描绘一个模拟世界中的各种角色与观点非常关键。

世界模拟项目中的环境通常会被专门设计，以反映所要模拟的情景，而智能体则会以不同画像与之匹配。与以协作为中心的问题求解系统不同，世界模拟系统在智能体管理与通信方面的方法更为多样，因为它们需要更真实地反映现实世界交互的复杂性与多样性。下面作者依次讨论不同领域中的模拟研究。

#### 原文标题：4.2.1 Societal Simulation

在社会模拟（societal simulation）中，LLM 多智能体模型被用于模拟社会行为，目的是探索潜在社会动力学与传播过程，测试社会科学理论，并用更真实的社会现象去填充虚拟空间和社区。

借助 LLM 的能力，拥有不同画像的智能体会开展大量交流，从而产生丰富的行为数据，用于深入的社会科学分析。

社会模拟的规模随着时间不断扩大，从较小、较亲密的设定逐步发展到更大、更复杂的系统。

- [Park et al., 2023] 引入生成式智能体（generative agents），在一个类似《模拟人生》的交互式沙盒环境中构造一个由 25 个 agent 组成的小型社区，允许终端用户通过自然语言与之互动。
- 与此同时，[Park et al., 2022] 开发了 Social Simulacra，构造一个由 1000 个 persona 组成的模拟社区。系统接收设计者对社区目标、规则与成员画像的设想，然后模拟发帖、回复、甚至反社会行为等多种活动。
- 在此基础上，[Gao et al., 2023a] 进一步构建了规模为 8563 和 17945 个智能体的巨大网络，分别模拟围绕“性别歧视”和“核能”主题展开的社会网络。

这些发展展示了近来研究中模拟环境规模与复杂性持续提升的趋势。作者还提到，近期一些研究进一步体现了多智能体系统复杂性的演进、LLM 对社会网络的影响，以及它们向社会科学研究中的融入。

#### 原文标题：4.2.2 Gaming

LLM 多智能体非常适合构建模拟游戏环境，让智能体在游戏中扮演不同角色。这种技术可以开发出可控、可扩展且动态变化的环境，并逼近人类在游戏中的互动方式，因此特别适合用来测试博弈论假设，例如推理、合作、说服、欺骗、领导等。

大多数由 LLM-MA 驱动的游戏模拟，都高度依赖自然语言通信，并在不同游戏设定中提供沙盒环境。

- [Akata et al., 2023] 借助行为博弈论，研究 LLM 在互动社会场景中的行为，特别是在重复囚徒困境和性别之战这类游戏中的表现。
- [Xu et al., 2023b] 借助 ChatArena 框架，让 LLM 参与像狼人杀这样的通信型游戏，并通过检索与对过往通信的反思进行能力提升，同时结合思维链（Chain-of-Thought）机制。
- [Light et al., 2023b] 探索 LLM 智能体参与 Resistance Avalon 的潜力，并提出 AVALONBENCH，作为一个综合性的游戏环境与基准，用于推动更高级的 LLM 与多智能体框架研究。
- [Wang et al., 2023c] 同样聚焦 LLM 智能体在 Avalon 中处理错误信息（misinformation）的能力，并提出 Recursive Contemplation（ReCon）框架，以增强 LLM 识别并对抗欺骗信息的能力。
- [Xu et al., 2023c] 将 LLM 与强化学习（RL）结合，开发用于狼人杀的策略语言智能体，并探索在动作空间和状态空间都不是预定义、而是自然语言表达的情况下如何使用 RL policy。
- [Mukobi et al., 2023] 设计了 Welfare Diplomacy，这是一种把零和桌游 Diplomacy 扩展为一般和博弈（general-sum）的设定，要求玩家在军事征服与国内福利之间取得平衡。该工作还提供了一个开源基准，旨在帮助提升多智能体 AI 系统的协作能力。
- 此外，[Li et al., 2023c] 在一个多智能体协作文本游戏中测试智能体的“心智理论（Theory of Mind, ToM）”，即推理他人隐藏心理状态的能力，这对人类社会互动、合作与沟通至关重要。
- [Fan et al., 2023] 系统评估了 LLM 作为理性玩家的能力，并发现即便在规则显式的游戏流程中，基于 LLM 的智能体仍可能在行动时忽视或修改那些本应更精细维护的信念。

#### 原文标题：4.2.3 Psychology

在心理学模拟研究中，与社会模拟类似，多个智能体被用来模拟具有不同特质与思维过程的人类。然而，与社会模拟不同，心理学中的一种做法是：**直接把心理学实验施加到这些智能体上**，然后用统计方法观察和分析它们的行为差异。在这种情况下，每个智能体是独立运行的，不与其他智能体互动，本质上代表不同个体。

另一种做法则更接近社会模拟：多个智能体之间相互交流和互动，再用心理学理论去理解和分析由此产生的行为模式。这样就可以研究个体心理特征如何影响群体行为与人际互动。

- [Ma et al., 2023] 研究将基于 LLM 的对话智能体用于心理健康支持时所带来的心理学影响与结果，强调必须从心理学角度慎重评估这类 agent 的使用。
- [Kovač et al., 2023] 提出 SocialAI school，一种用于创建互动环境的工具，用以模拟社会交互，并借鉴发展心理学来研究智能体如何获得、展示与演化诸如共同注意（joint attention）、沟通与文化学习等社会技能。
- [Zhang et al., 2023d] 研究具有不同人格特质与思维模式的 LLM agent 如何模拟从众（conformity）与多数规则（majority rule）等类人社会行为。作者认为，这种将心理学引入对 agent 协作机制的理解，为审视与提升多智能体系统提供了新的视角。
- [Aher et al., 2023] 提出图灵实验（Turing Experiments），用来评估大语言模型在多大程度上能够模拟人类行为的不同侧面。这些实验复现了心理学、经济学与社会学中的经典实验，用问答形式来模拟实验条件。它们还设计了一种提示词，通过改变名字来模拟不同个体的回答。结果表明，更大的模型通常更能真实复现人类行为，但在知识型任务中也会暴露出一种“过度准确（hyper-accuracy distortion）”失真现象。

#### 原文标题：4.2.4 Economy

LLM 多智能体之所以被用于模拟经济与金融交易环境，主要是因为它可以充当人类的隐式计算模型（implicit computational models of humans）。在这类模拟中，智能体会被赋予初始资源、信息以及预定义偏好，从而可以观察它们在经济与金融情境中的行为。这一点类似于经济学中的“经济人（homo economicus）”假设，即把人看作追求自身利益与财富的理性主体。

已有多项研究展示了 LLM-MA 在经济场景模拟中的多样化用途，包括：

- 宏观经济活动
- 信息市场
- 金融交易
- 虚拟小镇模拟

这些场景中，智能体往往处于协作式或辩论式、去中心化的环境里。

- [Li et al., 2023e] 使用 LLM 进行宏观经济模拟，借助 prompt engineering 驱动的智能体去模仿类人决策，从而比规则系统或其他 AI agent 更真实地呈现经济模拟。
- [Anonymous, 2023] 研究信息市场中的“买家检验悖论（buyer’s inspection paradox）”，发现当智能体在购买之前短暂获得信息访问权时，它们的决策质量与答案质量都得到改善。
- [Li et al., 2023g] 提出用于金融交易的 LLM-MA 框架，强调分层记忆系统、辩论机制以及个性化交易角色，从而增强决策稳健性。
- [Zhao et al., 2023] 利用基于 LLM 的智能体模拟一个拥有餐厅与顾客 agent 的虚拟小镇，并得到与社会学和经济学理论一致的洞见。

这些研究共同展示了 LLM 被用于多样经济模拟场景时的广泛潜力与研究进展。

#### 原文标题：4.2.5 Recommender Systems

LLM 多智能体在推荐系统中的用法，与它在心理学中的用法有些相似，因为这两个领域都会涉及人类的外在与内在因素，例如认知过程与人格特征。

一种用法是：把不同物品直接呈现给多个具有不同特质的基于 LLM 的智能体，再统计不同智能体的偏好。

另一种用法是：把**用户和物品都建模成智能体**，并把用户—物品之间的交流视作交互，从而模拟偏好传播。

为了弥合离线指标与真实世界推荐表现之间的差距，Agent4Rec 引入了一个基于 LLM-MA 的模拟平台。它用 MovieLens-1M 数据集初始化 1000 个生成式智能体，以模拟推荐环境中的复杂用户交互。Agent4Rec 表明，LLM 多智能体能够有效模仿真实用户偏好与行为，提供对过滤气泡（filter bubble）等现象的洞察，并帮助挖掘推荐任务中的因果关系。

在 Agent4Rec 中，智能体用来模拟用户，彼此之间并不通信。与之不同，[Zhang et al., 2023e] 把用户和物品都视为智能体，并让它们共同优化，以反映并调整真实世界中的交互偏差。该工作强调模拟用户—物品交互，并在智能体之间传播偏好，从而捕捉协同过滤（collaborative filtering）的本质。

#### 原文标题：4.2.6 Policy Making

与游戏和经济场景中的模拟类似，政策制定（Policy Making）也需要很强的决策能力来应对真实且动态复杂的问题。LLM 多智能体可以被用于模拟政策制定，例如：

- 模拟一个虚拟政府
- 模拟不同政策对不同社区的影响

这些模拟能够为政策如何形成、以及可能产生何种后果提供宝贵洞察，帮助决策者理解并预判决策影响。

[Xiao et al., 2023] 的研究聚焦于模拟一个乡镇水污染危机。它构造了一个位于岛屿上的小镇，包含不同人口结构的 agent、乡镇负责人和顾问。在这场水污染危机模拟中，作者详细分析了一个虚拟政府实体可能如何回应这类公共治理挑战，以及信息如何在社会网络中传播。

[Hua et al., 2023] 提出 WarAgent，用来模拟关键历史冲突，并为冲突解决与理解提供洞察，同时也可能帮助预防未来国际冲突。

#### 原文标题：4.2.7 Disease Propagation Simulation

利用 LLM 多智能体的社会模拟能力，也可以进行疾病传播模拟。

[Williams et al., 2023] 最近研究了 LLM-MA 在疾病传播模拟中的使用。研究表明，这些基于 LLM 的智能体能够较为准确地模拟人类在疾病暴发中的反应，例如病例数上升时采取自我隔离与居家隔离等行为。这些智能体的集体行为会呈现出疫情中典型的多轮波动模式，最终趋于地方性流行（endemic state）。令人印象深刻的是，它们的行为还会帮助削弱疫情曲线。

[Ghaffarzadegan et al., 2023] 也讨论了流行病传播模拟，并把问题拆成两部分：

- **机制模型（Mechanistic Model）**：表示病毒的信息传播或扩散机制；
- **决策模型（Decision-Making Model）**：表示智能体面对病毒时的决策过程。

## 本章小结

> [!note] 译者注（非原文）
> 这一章最重要的不是背论文，而是看出作者如何“按用途切分领域”：
> 1. **Problem Solving**：重点是多个 agent 如何分工合作，把任务做成。
> 2. **World Simulation**：重点是多个 agent 如何模拟复杂主体互动，把世界演出来。
> 3. 软件开发、具身智能、科学团队都更像“协作生产系统”。
> 4. 社会、游戏、心理、经济、政策、疾病传播更像“复杂系统模拟器”。
> 5. 同样叫 multi-agent，不同应用的真正目标函数可能完全不同：有的看任务成功率，有的看行为真实性，有的看涌现现象。

## 思考题

### 题目 1：为什么作者把“软件开发”和“社会模拟”放在同一篇综述里讨论？

> [!tip] 理解提示（非原文）
> 想想它们虽然表面差异很大，但都属于“多个 agent 共同参与一个复杂系统”的范畴。

> [!note] 译者注（非原文）
> 因为作者关注的不是应用表面，而是其底层系统结构：多个具有不同画像的 agent，在环境中彼此通信、获取反馈并协同演化。软件开发和社会模拟虽然目标不同，但都能被同一组分析维度统一描述。

### 题目 2：问题求解型多智能体与世界模拟型多智能体最核心的差别是什么？

> [!warning] 易混点（非原文）
> 不要只回答“一个更偏工程，一个更偏社会科学”，那还不够。

> [!note] 译者注（非原文）
> 更本质的差别在于目标函数不同：问题求解系统更关心“任务是否完成、结果是否更优”；世界模拟系统更关心“行为是否真实、系统是否能呈现复杂互动与涌现现象”。因此，它们的环境构造、角色设置、通信组织和评测方式都会不同。
