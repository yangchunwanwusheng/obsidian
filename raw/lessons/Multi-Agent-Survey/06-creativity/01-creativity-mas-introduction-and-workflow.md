# LLM 多智能体综述中文精读（第 6 篇-01）：导论与工作流

> 本篇覆盖 **Section 1 Introduction、Section 2 MAS Workflow and Proactivity**。核心任务是理解：为什么创造力是 LLM-MAS 中被忽视的维度？以及多 agent 创意系统的流程和主动性机制是怎么设计的？

## 本章导读

> [!note] 译者注（非原文）
> 这一章有两个核心任务：
> 1. 建立"创造力在 MAS 中为什么重要"的问题意识——之前的综述谈了协作、架构、应用，但都没把创造力当成独立分析对象；
> 2. 给出分析创造力的第一个维度——系统的工作流和 agent 的主动性水平。
>
> 值得一提的是，这篇论文来自台湾大学李宏毅（Hung-yi Lee）和陳縕儂（Yun-Nung Chen）课题组，在 EMNLP 2025 正式发表。台湾大学在 NLP 和语音处理领域有深厚积累，这篇综述延续了该组对多智能体系统前沿的关注。

## 正文翻译

### 原文标题：1 Introduction

#### 1.1 研究动机与背景

近年来，基于大语言模型（LLM）的多智能体系统（MAS）研究经历了爆炸式增长。研究者们已经展示了 MAS 在复杂推理、软件开发、科学发现、社会模拟等众多领域中的卓越表现。然而，作者指出一个重要的空白：**现有综述工作几乎完全忽略了"创造力"这一维度**。

作者认为，创造力是人类集体智能的核心特征之一。在人类社会中，许多最具创新性的成果都来自团队的协作创作——从科学研究到艺术创作，从产品设计到政策制定。当我们将多个 LLM agent 组合在一起时，一个自然的问题是：**这些 agent 的集体创造力能否超越单 agent 的上限？**

作者进一步指出，创造力在 MAS 中并不是一个自然涌现的属性——它需要通过特定的系统设计来实现。具体来说，创造力 MAS 需要在三个层面进行精心设计：

1. **工作流设计**：多 agent 之间的任务如何分配、信息如何流转、决策如何做出；
2. **技术选择**：用什么具体技术机制来激发和组合 agent 的创意产出；
3. **评估方法**：如何客观、可重复地衡量一个 MAS 的创造力水平。

这三个层面构成了本综述的核心分析框架。

#### 1.2 创造力在 MAS 中的独特价值

作者从四个角度论证了创造力作为 MAS 独立分析维度的重要性：

**第一，创造力是 MAS 区别于单 agent 的关键增益之一。** 单 agent 即使参数规模再大，其创意产出仍然受限于单条推理链。多 agent 系统通过让不同"人格"、不同知识背景的 agent 相互碰撞，有潜力产生更多样、更新颖的输出。

**第二，创造力任务的评估范式与通用任务不同。** 在通用 NLP 任务中，我们有明确的 ground truth 或 human preference 作为评判标准。但在创造力任务中，"好"的标准本身就是多维的、主观的、甚至存在争议的。这需要一套不同的评估方法论。

**第三，创造力 MAS 面临的挑战具有独特性。** 例如：如何在"多样性"和"质量"之间取得平衡？如何避免 agent 之间的创意同质化？当 agent 产生冲突意见时如何处理？这些问题在通用 MAS 中不明显，但在创造力场景中至关重要。

**第四，创造力 MAS 的应用场景正在快速扩张。** 从创意写作、广告文案生成、产品命名，到音乐创作、游戏剧情设计、影视剧本协作——越来越多的实际应用需要 MAS 具备创造性产出能力。

#### 1.3 本文的贡献与论文组织

作者总结了本文的四个主要贡献：

1. **首次系统综述**：第一篇专门聚焦 LLM-MAS 中创造力的综述论文；
2. **分类学框架**：提出结构化的分析框架，从 Workflow、Techniques、Persona、Evaluation 四个维度组织现有工作；
3. **评估方法整理**：系统整理创造力评估的客观指标和主观方法；
4. **开放问题识别**：明确当前领域面临的挑战和未来研究方向。

论文结构如下：
- Section 2：介绍 MAS 工作流和 agent 主动性
- Section 3：介绍 MAS 中用于增强创造力的技术
- Section 4：讨论 agent persona 和画像设计
- Section 5：系统整理创造力评估方法
- Section 6：讨论挑战与未来方向
- Section 7：结论

> [!tip] 理解提示（非原文）
> 注意这篇综述与第 3 篇（协作机制）的关系：第 3 篇把协作机制分成 type / strategy / structure / coordination 四个维度，关注的是"怎么让 agent 一起工作"；第 6 篇则聚焦于"怎么让 agent 一起工作得更有创造力"，维度是 workflow / technique / persona / evaluation。两篇框架互补但不重叠。

### 原文标题：2 MAS Workflow and Proactivity

> [!note] 译者注（非原文）
> Section 2 是整个框架的第一个维度——工作流与主动性。作者先把多 agent 创意系统的运行流程抽象为三个阶段，然后引入"主动性谱系"来刻画 agent 在系统中的行为模式。这两个概念是理解后续所有技术和评估方法的基础。

#### 2.1 MAS 创意工作流的三阶段模型

作者将创造力 MAS 的典型工作流抽象为三个递进阶段：

##### Phase 1: Planning（规划阶段）

在规划阶段，系统需要确定：
- **任务分解**：将创意任务拆分为可分配的子任务。例如，一个"写一首关于秋天的诗"的任务可能被分解为：意象收集、情感基调确定、初稿生成、韵律调整、终稿润色；
- **角色分配**：确定哪些 agent 负责哪些子任务，根据每个 agent 的 persona 和能力进行匹配；
- **流程定义**：确定子任务之间的依赖关系和执行顺序（串行 / 并行 / 条件分支）。

规划阶段的核心挑战在于：创意任务本质上具有高度的不确定性，很难在规划阶段就完全预见所有需求。因此，好的规划需要保留一定的灵活性。

##### Phase 2: Process（执行阶段）

在执行阶段，各 agent 按照规划执行各自的子任务。这一阶段的关键机制包括：

- **生成**：agent 根据其角色和任务描述生成创意输出；
- **交互**：agent 之间进行信息交换、反馈、讨论。交互模式可以是回合制的（turn-based），也可以是并发的（concurrent）；
- **记忆与状态管理**：系统需要维护 agent 的生成历史、交互记录和中间产物，供后续阶段参考。

执行阶段是创造力真正产生的地方。作者强调，这一阶段的设计直接影响创意产出的多样性（diversity）和新颖性（novelty）。

##### Phase 3: Decision Making（决策阶段）

在决策阶段，系统需要从多个候选输出中选择最终结果，或对候选结果进行整合。常见决策机制包括：

- **投票（Voting）**：多个 agent 对候选输出进行投票；
- **评判（Judging）**：由一个专门的评判 agent 评估所有候选；
- **共识（Consensus）**：agent 通过讨论达成共识；
- **级联选择（Cascaded Selection）**：多轮筛选，每轮淘汰部分候选。

作者指出，决策阶段是创意质量控制的关键环节。一个好的决策机制需要在"保留创意多样性"和"确保输出质量"之间取得平衡。

> [!tip] 理解提示（非原文）
> 这三阶段模型（Planning → Process → Decision Making）实际上是一个"发散-收敛"循环：规划阶段设置约束，执行阶段发散探索，决策阶段收敛选择。这与经典的设计思维（Design Thinking）中的 diverge-converge 模式高度一致。

#### 2.2 主动性谱系（Proactivity Spectrum）

作者提出了一个非常重要的概念——agent 的主动性谱系，从完全被动（reactive）到高度主动（proactive）：

##### Reactive（被动响应）端

处于这一端的 agent 只有在接收到明确指令或触发条件时才会行动。典型特征：
- 等待人类或其他 agent 的指令；
- 不对任务进行自主扩展或重新解释；
- 输出严格限定在给定任务范围内。

被动 agent 的优点是可控性强、输出可预测；缺点是缺乏创造性突破的可能性。

##### Proactive（主动发起）端

处于这一端的 agent 会主动：
- **提出新想法**：在当前任务之外，主动建议新的创意方向；
- **质疑既有方案**：对已有设计提出批判性意见和替代方案；
- **自主扩展任务**：在完成任务的基本要求后，主动增加额外的创意内容；
- **预见需求**：预判后续步骤可能需要的创意输入，提前准备。

作者指出，高度主动的 agent 虽然能带来更多创意可能，但也带来了控制难度增加、输出不确定性升高等挑战。

##### 中间状态

在 reactive 和 proactive 之间，存在多种中间状态。例如：
- **条件性主动（Conditionally Proactive）**：只在特定条件下（如遇到瓶颈、检测到质量不足时）才主动介入；
- **建议性主动（Suggestively Proactive）**：提出建议但不强制执行，最终决策权留给人类或其他 agent；
- **协商性主动（Negotiatively Proactive）**：在提出主动行动前，先与其他 agent 或人类协商。

> [!tip] 理解提示（非原文）
> 主动性谱系的设计实际上反映了人类团队管理中"赋权（empowerment）"的问题——你给团队成员多大自主权？自主权太高可能导致混乱，自主权太低则扼杀创造力。MAS 的主动性设计面临同样的权衡。

#### 2.3 工作流与主动性的协同

作者最后讨论了工作流设计如何与主动性水平协同运作：

1. **规划阶段与主动性**：在规划阶段，如果 agent 具有较高的主动性，它们可以参与任务分解和角色分配过程，提出不同的规划方案供选择。这是一种"元级创意"——不是对任务本身的创意，而是对"如何完成任务"的创意。

2. **执行阶段与主动性**：在执行阶段，主动性直接决定了创意产出的广度和深度。高主动性的 agent 会在生成过程中不断自我质疑、自我修正、探索替代方案。

3. **决策阶段与主动性**：在决策阶段，主动性的一个关键体现是 agent 是否能够挑战已有的评判标准。例如，一个高主动性的评判 agent 可能不仅评估候选输出，还会主动提出"我们是否应该重新定义什么是好的创意"。

作者用一个简洁的对比总结了这个维度的价值：

> 一个完全被动的 MAS 就像一个严格按菜谱做菜的厨房——每个步骤都按计划执行，但做出来的永远是同一道菜。而一个高度主动的 MAS 则像一个有创造力的厨师团队——他们会根据食材状态调整配方、尝试新的搭配、甚至创造全新的菜品。

> [!note] 译者注（非原文）
> 这个厨房类比非常精准。它同时揭示了为什么"有创造力"往往是更可取的——但也暗示了为什么很多实际系统仍然选择低主动性：因为稳定性和可复现性在很多场景下优先级更高。

## 思考题

### 题目 1：为什么说"创造力"应被视为 LLM-MAS 的独立分析维度？

> [!tip] 理解提示（非原文）
> 想想通用 MAS 和创造力 MAS 在目标、评估标准、系统设计上有什么本质区别？

> [!success]- 参考答案（非原文）
> 因为创造力任务与通用任务有三个根本区别：（1）没有明确的 ground truth，"好"与"坏"的评判是多维且主观的；（2）需要在多样性（diversity）和质量（quality）之间主动平衡，而通用任务通常只追求质量；（3）agent 的"人格"和"风格"在创造力任务中是核心变量，而非可忽略的噪声。

### 题目 2：为什么作者要把 agent 的主动性设计为一个"谱系"而非二分法？

> [!tip] 理解提示（非原文）
> 想想如果只有"全被动"和"全主动"两种选择，会丢失哪些有意义的中间设计？

> [!success]- 参考答案（非原文）
> 因为在实际系统中，最优的主动性水平通常是场景相关的。二分法（要么完全被动要么完全自主）会迫使设计者做出极端选择，忽略大量有价值的中间配置——例如"条件性主动"（只在检测到质量不足时介入）或"建议性主动"（提出建议但不强制）——这些配置在很多场景下比两个极端更实用。
