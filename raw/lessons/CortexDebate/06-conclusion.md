# CortexDebate — 结论与局限性 (Section 6: Conclusion & Limitations)

> 本篇翻译论文第 6 节 Conclusion 和 Limitations 的内容。

## 正文翻译

### 6 结论

在本文中，我们提出了一种新的 MAD 方法——"CortexDebate"，以提升多智能体交互系统的推理能力。具体来说，我们的 CortexDebate 在参与的 LLM 智能体之间建立了稀疏辩论图，减少了 LLM 智能体的输入信息负担。此外，通过整合 McKinsey Trust Formula，我们提出的 MDM 模块进行可信评估，逐步优化辩论图，使辩论过程更加平等、深入和有效。

由于上述设计，我们的方法缓解了现有 MAD 系统面临的两大主要问题（即过长的输入上下文和过度自信导致的不平等辩论），并在各种典型任务上展现出优于各种最先进 MAD 方法的性能。

在未来，我们计划继续探索 CortexDebate 在大规模辩论和复杂任务（即领域专家系统）中的潜力。

### 局限性

尽管 CortexDebate 展现了令人印象深刻的性能，我们承认它存在两个主要局限：

**局限 1：效率与成本。** 作为一种多智能体辩论方法，与单智能体方法相比，在解决任务时不可避免地会出现效率下降和成本增加。

**局限 2：受限于底层 LLM 的推理能力。** 尽管取得了成功，LLM 智能体的推理能力仍然是限制 CortexDebate 性能的重要因素。虽然 CortexDebate 改进了 LLM 智能体之间的辩论策略，但由于 LLM 智能体推理能力不足，错误仍可能发生。

> [!note] 译者注（非原文）
> 这两个局限性都是 MAD 方法固有的通病：
> 1. 多智能体必然需要多次推理调用，成本是单智能体的 n 倍。不过 CortexDebate 的稀疏图在一定程度上缓解了这个问题——不是每次都与所有人辩论，减少了输入 token 数。
> 2. "垃圾进，垃圾出"——如果所有底层 LLM 都缺乏某个领域的推理能力，再好的辩论策略也无法凭空产生正确答案。辩论的优势在于"集合智慧"，但当所有智能体都不擅长某个问题时，集合智慧也无济于事。

### 论文核心 Prompt 设计（附录 F）

论文提供了 CortexDebate 使用的两种 Prompt：

**初始答案生成 Prompt：**

```
Question: {问题描述}
Please think it step by step and generate an answer and an explanation for your answer.
Also, evaluate how confident you are that your answer is correct.
Your confidence score should between 0 and 1.
The format of your answer must be:
Answer: (...)
Explanation: (...)
Confidence Score: (...)
```

**答案再生成 Prompt：**

```
Question: {问题描述}
There are some answers generated by other LLM agents:
One LLM agent answer: {答案}
One LLM agent answer: {答案}
... ...
Using these answers as additional information, please generate a new answer
and an explanation for your answer.
Also, evaluate how confident you are that your answer is correct.
Your confidence score should between 0 and 1.
The format of your answer must be:
Answer: (...)
Explanation: (...)
Confidence Score: (...)
```

> [!tip] 理解提示（非原文）
> Prompt 设计相对简洁。关键点在于：答案再生成时，智能体只接收稀疏图中相连的对手的答案（不是所有人的答案）。这减少了输入信息量，同时也更聚焦——只看"对自己有帮助的"对手的观点。

### 基线方法详细介绍（附录 G）

**Multi-agent Voting (MaV)**：每个智能体独立生成答案，通过多数投票决定最终预测。

**ChatEval**：使用额外的 LLM 智能体在每轮辩论中总结辩论结果。总结文本作为下一轮辩论的补充信息输入给每个智能体。

**Multi-LLM Debate (MLD)**：首先每个 LLM 生成答案。然后每个 LLM 阅读并批评其他智能体的答案，生成新答案。此步骤重复多次。最终通过多数投票获得答案。

**RECONCILE**：给定问题，每个 LLM 首先生成答案和不确定性。然后所有智能体进入多轮辩论，每轮基于上一轮所有其他智能体的答案生成修正答案和新的不确定性。最终通过多数投票获得答案。

**Peer Review Debate (PRD)**：类似 RECONCILE，但采用同行评审策略——每个智能体的答案由其他智能体评分。

**GroupDebate (GD)**：将智能体分为若干辩论组，每组进行内部辩论。内部辩论后，每组的结果被汇总放入共享池。每组从池中检索所有组的辩论总结，作为下一轮所有智能体的输入。

**Neighbor Debate (ND)**：LLM 仅与其邻居辩论（拓扑固定）。

## 整体评价与思考

### 论文的亮点

1. **生物学启发的类比非常有说服力**：将 LLM 智能体对应为皮层区域，MDM 对应为白质，稀疏辩论图对应为皮层间稀疏网络，这个类比既直观又有理论支撑。

2. **McKinsey Trust Formula 的引入是关键创新**：将社会学中的信任评估公式适配到 MAD 场景，同时考虑个体能力和协作效果，有效解决了过度自信问题。

3. **实验非常全面**：4 类任务、8 个数据集、多种基线方法、100 样本和 1000 样本两个规模、详细的消融实验和深入分析。

4. **"少即是多"的验证**：减少辩论对手反而提升性能，这一反直觉的发现很有价值。

### 论文的不足

1. **未在闭源大模型上实验**：仅使用 7B-9B 开源模型，未验证在 GPT-4、Claude 等更大模型上是否仍有显著提升。

2. **缺乏效率与成本的量化分析**：虽然展示了输入 token 数减少，但未提供总 API 调用成本、端到端运行时间等实用性指标。

3. **稀疏图动态性的分析不够深入**：论文展示了动态优化稀疏图的效果，但未分析辩论图在多轮辩论中的演化过程——哪些连接被保留？哪些被移除？为什么？

4. **对"平等辩论"的衡量标准有限**：论文主要通过共识比例和分数变化来论证辩论的"平等性"，但缺乏对每个智能体话语权变化的具体分析。

## 关键要点回顾

- CortexDebate 通过稀疏辩论图和 MDM 模块有效解决了 MAD 的两大核心问题
- 两个主要局限：效率/成本和受限于底层 LLM 能力
- 未来方向：大规模辩论和领域专家系统
- Prompt 设计简洁，易于复现
