# T5

> 将所有 NLP 任务统一为"文本到文本"生成问题的 Encoder-Decoder 模型，以 Span Corruption 为预训练目标。

## 基本信息

| 字段 | 内容 |
|------|------|
| 全称 | Text-to-Text Transfer Transformer |
| 论文 | [Exploring the Limits of Transfer Learning with a Unified Text-to-Text Framework](https://arxiv.org/abs/1910.10683) |
| 团队 | Google |
| 年份 | 2019 |
| 架构类型 | Encoder-Decoder（编码器双向注意力 + 解码器单向注意力 + 交叉注意力） |

## 概述

T5 保留完整的 Transformer 编码器-解码器架构，核心思想是**将所有 NLP 任务统一为文本生成问题**——无论分类、翻译、摘要还是问答，输入和输出都是文本字符串。预训练采用 **Span Corruption**：随机遮蔽连续 token 跨度，要求模型生成被遮蔽的内容。

## 核心创新

### Text-to-Text 统一框架

传统上不同 NLP 任务需要不同的输出头（分类用全连接层、问答用 span 预测、NER 用 CRF）。T5 的做法是统一格式：

```
翻译任务：输入 "translate English to French: Hello" → 输出 "Bonjour"
分类任务：输入 "sentiment: This movie is great!" → 输出 "positive"
摘要任务：输入 "summarize: [长文...]" → 输出 "[摘要...]"
```

所有任务共享同一个序列到序列的生成目标，无需为不同任务设计专用输出层。

### Span Corruption 预训练

不同于 BERT 的逐 token 遮蔽，T5 随机遮蔽连续的 token 跨度（span），用哨兵 token（sentinel token）标记位置，要求解码器按顺序生成被遮蔽的内容。这种方式更接近生成任务的实际需求。

## 与 BERT / GPT 的对比

| 维度 | T5 | BERT | GPT |
|------|-----|------|-----|
| 架构 | Encoder-Decoder | Encoder-Only | Decoder-Only |
| 编码器注意力 | 双向 | 双向 | — |
| 解码器注意力 | 单向 + 交叉注意力 | — | 单向 |
| 预训练任务 | Span Corruption | MLM + NSP | 下一个 token 预测 |
| 生成方式 | 自回归（编码后解码） | 非自回归 | 自回归 |
| 典型应用 | 翻译、摘要、改写 | 分类、NER、问答 | 续写、对话、代码 |
| 典型参数量 | 1 亿 ~ 110 亿 | 1 亿 ~ 3 亿 | 70 亿 ~ 1.8 万亿 |

T5 像"翻译专家"——先完整阅读原文（编码器），理解后再用目标语言表达（解码器），兼顾理解与生成。
