# RNN 体系 vs Transformer：序列建模的范式之争

> 从"边读边记"到"全员圆桌"，两种序列建模架构的根本差异与各自适用场景。

## 核心对比

| 维度 | RNN / LSTM | Transformer |
|------|-----------|-------------|
| **计算方式** | 逐 token 顺序处理 | 所有 token 同时处理（矩阵运算） |
| **并行性** | 无（$h_t$ 依赖 $h_{t-1}$，串行链） | 完全并行（$QK^T$ 一次算完所有位置对） |
| **梯度路径** | 长度 = 序列长度 $T$（经过 $T$ 步连乘） | 长度 = 常数（与序列长度无关） |
| **感受野** | 逐步扩展，远距离需 $T$ 步传递 | 全局：每个位置一步直达所有其他位置 |
| **长程依赖** | 有效记忆约 5-10 步（RNN），100-300 步（LSTM） | 距离无关，第 1 个和第 1000 个 token 的交流成本相同 |
| **训练速度** | 慢（GPU 利用率低，串行瓶颈） | 快（矩阵运算天然并行，充分利用 GPU） |
| **计算复杂度** | $O(n)$（串行） | $O(n^2)$（可并行） |
| **参数效率** | 参数少，适合小模型 | FFN 占参数主体（约 65%），参数量大 |
| **序列长度限制** | 理论无限，实际受梯度消失约束 | 受 $O(n^2)$ 注意力矩阵约束 |
| **代表模型** | RNN、LSTM、GRU、Seq2Seq | GPT、BERT、LLaMA、ViT |

## 发展脉络

```
RNN (1986)
  │  问题：梯度消失，有效记忆 5-10 步
  ▼
LSTM (1997)
  │  改进：门控机制缓解梯度消失
  │  遗留：顺序依赖（无法并行）
  ▼
Seq2Seq (2014)
  │  改进：编码-解码范式
  │  问题：固定维度上下文向量的信息瓶颈
  ▼
Seq2Seq + Attention (Bahdanau, 2014)
  │  改进：动态聚焦，打破信息瓶颈
  │  遗留：编码器/解码器仍是 RNN，顺序依赖
  ▼
Transformer (Vaswani, 2017)
     核心：Attention Is All You Need
     — 只用 Attention，抛弃递归
     — 并行 + 全局感受野 + 健康梯度
```

## RNN 的优势场景

尽管 Transformer 已成为主流，RNN 体系在以下场景仍有不可替代的价值：

| 场景 | 原因 | 典型应用 |
|------|------|---------|
| **资源受限设备** | 参数少、内存占用小，适合边缘设备部署 | 嵌入式语音识别、可穿戴设备 |
| **在线/流式处理** | 天然逐个处理输入，无需等待完整序列 | 实时语音转写、在线翻译 |
| **超长序列** | $O(n)$ 复杂度，不受序列长度平方增长制约 | 超长时序预测、连续传感器数据 |
| **因果推理** | 严格的时间因果性，不"偷看"未来信息 | 时间序列预测、在线控制 |
| **小数据场景** | 参数少，不易过拟合 | 低资源语言的序列标注 |

## Transformer 的优势场景

| 场景 | 原因 | 典型应用 |
|------|------|---------|
| **大规模预训练** | 并行训练效率极高，可利用海量 GPU 集群 | GPT、BERT、LLaMA |
| **GPU 密集型任务** | 矩阵运算完美适配 GPU 架构 | 大语言模型训练与推理 |
| **需要全局上下文** | Self-Attention 天然提供全局感受野 | 长文档理解、代码生成 |
| **多模态任务** | Attention 可统一不同模态的信息交互 | 视觉-语言模型（CLIP、Flamingo） |
| **迁移学习** | 预训练-微调范式成熟 | BERT fine-tuning、GPT prompting |

## 关键 trade-off 总结

Transformer 用 $O(n^2)$ 的计算复杂度换取了三个核心优势：**并行性 + 全局感受野 + 健康梯度路径**。在 GPU 时代，这个交换非常划算——并行的矩阵运算速度远超串行的逐步计算。只有当序列长度 $n$ 非常大时（如处理整本书），$O(n^2)$ 才会成为真正的瓶颈。

> 核心洞察：RNN 的顺序依赖是**架构本质**的限制（数学定义 $h_t = f(h_{t-1}, x_t)$ 决定），不是实现层面能绕过的问题。
