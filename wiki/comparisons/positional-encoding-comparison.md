# 位置编码方案对比：从正弦编码到 RoPE 与 ALiBi

> Self-Attention 是置换等变的——它不知道"谁在第几位"。位置编码方案决定了 Transformer 如何感知序列顺序，不同方案在外推能力、计算成本和适用场景上差异显著。

## 核心对比

| 维度 | 正弦编码（Sinusoidal） | 可学习编码（Learned） | RoPE（旋转位置编码） | ALiBi（线性偏置） |
|------|---------------------|--------------------|--------------------|--------------------|
| **提出时间** | 2017 | 2018 | 2021 | 2022 |
| **参数量** | $O(1)$（零参数，公式生成） | $O(n_{max} \times d)$（查表） | $O(1)$（零参数） | $O(h)$（每头一个斜率） |
| **外推能力** | 强（数学函数，任意长度可生成） | 弱（超训练长度即失效） | 强（相对位置旋转不变） | 强（线性偏置天然泛化） |
| **表达灵活性** | 受限于正弦函数形式 | 可学任意复杂关系 | 受限于旋转矩阵结构 | 受限于线性衰减形式 |
| **计算成本** | 需计算三角函数 | 查表 $O(1)$ | 需计算旋转矩阵 | 加法偏置，极低 |
| **位置类型** | 绝对位置（但具有线性性质） | 绝对位置 | 相对位置（天然） | 相对位置（天然） |
| **代表模型** | 原始 Transformer | BERT、GPT-1/2 | LLaMA、GLM-4、PaLM | BLOOM、MPT |

## 各方案核心思想与优缺点

### 1. 正弦位置编码（Sinusoidal）

**核心思想**：用不同频率的 $\sin/\cos$ 函数生成位置向量，低频编码大尺度位置，高频编码精细位置。

$$PE_{(pos, 2i)} = \sin\left(\frac{pos}{10000^{2i/d_{model}}}\right), \quad PE_{(pos, 2i+1)} = \cos\left(\frac{pos}{10000^{2i/d_{model}}}\right)$$

**关键性质**：存在线性变换 $M$ 使得 $PE_{pos+k} \approx M^k \cdot PE_{pos}$——知道某位置的编码可推算出相对位置的编码。

| 优点 | 缺点 |
|------|------|
| 零参数，任意长度可用 | 表达能力受限于正弦函数 |
| 天然外推（连续数学函数） | 无法针对任务调整 |
| 不同频率捕捉不同尺度关系 | GPU 计算三角函数有开销 |

### 2. 可学习位置编码（Learned）

**核心思想**：将位置编码当作可训练参数 $P \in \mathbb{R}^{n_{max} \times d_{model}}$，通过反向传播学习最优编码。

| 优点 | 缺点 |
|------|------|
| 理论上可学任意复杂位置关系 | 需预设最大长度，超长即失效 |
| 在充足数据下通常表现更好 | 参数量大（$n_{max} \times d$） |
| 推理时查表，计算成本极低 | 位置关系离散，泛化能力差 |

### 3. RoPE（Rotary Position Embedding）

**核心思想**：不把位置编码加到词嵌入上，而是对 Query 和 Key 施加旋转变换，使 Attention Score 只依赖相对位置。

$$(R_{pos}^Q q)^T (R_{pos'}^K k) = q^T R_{pos-pos'} k$$

这意味着 attention score 只依赖 $pos - pos'$（相对距离），不依赖绝对位置。

| 优点 | 缺点 |
|------|------|
| 天然编码相对位置 | 实现比加法编码复杂 |
| 长序列外推能力强 | 旋转矩阵计算有一定开销 |
| 被主流大模型广泛验证 | 远距离衰减可能过快 |

### 4. ALiBi（Attention with Linear Biases）

**核心思想**：不修改嵌入，直接在 attention score 上加一个与相对距离成正比的线性偏置。

$$score_{ij} = (q_i \cdot k_j) + |i - j| \cdot m$$

其中 $m$ 是预设斜率，不同注意力头用不同斜率。

| 优点 | 缺点 |
|------|------|
| 实现极简（只加一个偏置） | 表达能力受限于线性形式 |
| 完全不需要位置编码参数 | 可能不如 RoPE 灵活 |
| 长序列泛化极好 | 主流采用度不如 RoPE |

## 选型指南

| 场景 | 推荐方案 | 理由 |
|------|---------|------|
| **大规模预训练（10B+ 参数）** | RoPE | LLaMA 等验证过，相对位置编码效果最好 |
| **中等规模、数据充足** | 可学习编码 | BERT 路线成熟，效果稳定 |
| **需要超长序列外推** | ALiBi 或 RoPE | 两者都有优秀的长序列泛化能力 |
| **资源受限、小模型** | 正弦编码 | 零参数，不增加模型大小 |
| **快速原型/实验** | 可学习编码 | 实现最简单，PyTorch 内置支持 |

> 趋势：现代大模型（LLaMA、GLM-4、Mistral）已几乎统一采用 RoPE，可学习编码在 BERT 类模型中仍然主流，ALiBi 作为长序列方案的有力竞争者。
