# Mixture of Experts（混合专家模型）

> 通过稀疏激活机制，用少量实际计算量驱动庞大的参数空间，实现"小成本、大容量"的模型架构。

## 概述

Mixture of Experts（MoE，混合专家模型）是 Transformer 架构中一种重要的效率优化策略。其核心思想是：将传统 Transformer 中全连接的前馈网络（FFN）替换为一组并行的"专家"网络，每次推理时只激活其中一小部分。这使得模型的总参数量可以非常大（从而拥有更高的知识容量），但每个 token 的实际计算量却保持在与小模型相当的水平。

MoE 在大语言模型领域的代表性应用包括 Mixtral 8x7B（8 个专家中激活 2 个）以及推测被 GPT-4 采用的 MoE 架构。

## 核心机制：Router + Expert

MoE 层替代了标准 Transformer 中的 FFN 层，由两个核心组件构成：

- **Router（路由器）**：一个轻量的门控网络（通常是简单的线性层 + Softmax），接收当前 token 的隐藏状态作为输入，输出该 token 应被分配给哪些专家的概率分布。
- **Expert（专家）**：多个独立的 FFN 网络，每个专家拥有自己的参数。对于每个输入 token，Router 选择 top-k 个专家（通常 k=1 或 k=2），将 token 以加权方式发送给这些专家。

```
输入 token → Router 计算概率分布
           → 选择 top-k 个专家
           → 专家分别计算输出
           → 加权求和得到最终结果
```

以 Mixtral 8x7B 为例：总参数量 46.7B（8 个各约 7B 的专家），但每个 token 仅激活 top-2 专家，实际激活参数约 12B，计算量接近一个 12B 的 Dense 模型。

## 与 Dense Transformer 的关键区别

| 特性 | Dense Transformer | MoE Transformer |
|------|------------------|-----------------|
| **激活方式** | 每个 token 激活全部参数 | 每个 token 仅激活部分专家 |
| **总参数量** | 等于激活参数量 | 远大于实际激活参数量 |
| **每 token 计算量** | 与总参数量成正比 | 仅与激活的专家参数量成正比 |
| **知识容量** | 受限于计算预算 | 可大幅扩展而不等比例增加计算 |
| **推理吞吐量** | 较低 | 较高（相同计算量下可服务更多请求） |

**一句话总结区别**：Dense 模型是"全员参与"，MoE 模型是"按需调度"。

## 优点

- **更高的参数效率**：在不增加推理计算量的前提下，大幅扩展模型的总参数量和知识容量
- **更快的训练和推理**：相对于同等总参数量的 Dense 模型，训练和推理速度快得多
- **更好的任务特化**：不同专家可能自然学会处理不同类型的输入，实现隐性的任务分工

## 挑战

- **负载均衡问题**：Router 可能倾向将大部分 token 分配给少数几个专家（"赢家通吃"），导致其他专家得不到训练。解决方案包括辅助损失函数（auxiliary loss）和容量因子（capacity factor）等
- **显存开销**：虽然每次只激活部分专家，但所有专家的参数都必须加载到显存中，显存占用等于全部专家参数量之和
- **分布式训练复杂度**：专家分布在多张 GPU 上时，需要精细的通信调度（All-to-All 通信），工程实现难度高
- **训练不稳定性**：稀疏激活模式可能导致某些专家训练不足，影响整体性能

## 要点

- MoE 通过 Router 选择性激活 Expert 网络，实现"大参数量、小计算量"的目标
- 稀疏激活是 MoE 与 Dense Transformer 的本质区别——并非所有参数都参与每次前向传播
- 负载均衡和显存开销是 MoE 架构的两个核心工程挑战
- MoE 是当前大模型效率优化的主流方向之一，GPT-4 等顶级模型据推测采用了 MoE 架构
