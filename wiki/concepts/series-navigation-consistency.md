# 系列课程导航一致性检查

> 一句话摘要：系列课程的章节顺序、导航链接、路线路图必须保持一致，否则会严重误导读者学习路径。

## 概述

在多章节并行写作的学习系列中，容易出现"路线图描述的顺序"与"实际文件顺序/内容"不一致的问题。这种不一致会导致读者按错误路径学习，破坏知识的递进性。

## 核心检查清单

在创建或更新系列课程后，**必须**执行以下检查：

### 1. 路线图一致性
- [ ] 概览页的"项目时间线/路线图"是否与实际文件顺序一致
- [ ] 概览页的"学习路线表"中每章的主题、核心内容、难度是否与实际文件匹配
- [ ] 概览页的"下一步学习"链接是否指向正确的第一篇后续课程

### 2. 逐章导航一致性
- [ ] 每章末尾的"下一步学习"链接中的 **文件名** 是否与实际文件匹配
- [ ] 每章末尾的"下一步学习"链接中的 **显示标题** 是否与目标文件的标题一致
- [ ] 每章的 `prerequisites` 中引用的前置章节链接是否正确
- [ ] 每章的 frontmatter `series.part` 编号是否连续且无跳号

### 3. 双向导航完整性
- [ ] 是否有"上一章"链接（如果系列支持双向导航）
- [ ] 系列导航块中的所有 wikilink 都能正确跳转
- [ ] 系列的最后一章是否明确标注"系列结束"而非指向不存在的下一章

## 错误案例：LLM-Training 系列（2026-04-22 修复）

### 根因分析

并行子智能体写作时，各子智能体**独立生成导航链接**，但：
1. 概览页的路线图是最初规划的顺序（分词器→架构→数据工程...）
2. 实际写作时调整了顺序（数据工程→分词器→架构...）
3. 各子智能体根据"旧的规划"而非"实际的文件名"生成导航链接
4. 缺乏最终的"导航一致性"检查步骤

### 具体错误清单

| 位置 | 错误链接 | 正确链接 |
|------|---------|---------|
| 01 §4.4 路线图 | 第2章=分词器 | 第2章=数据工程 |
| 01 §5 学习路线表 | 全部10章描述与实际不符 | 需全部重写 |
| 01 末尾 | `02-tokenizer` | `02-data-engineering` |
| 02 末尾 | `03-tokenization` | `03-tokenizer-design` |
| 03 末尾 | `04-model-architecture` | `04-modern-llm-architecture` |
| 04 末尾 | `05-training-pipeline` | `05-distributed-training` |
| 06 末尾 | `07-finetuning-and-alignment` | `07-pretraining-practice` |
| 07 末尾 | `08-sft-and-alignment` | `08-supervised-finetuning` |
| 08 末尾 | `09-reinforcement-learning-alignment` | `09-rlhf-dpo` |

## 预防措施

在 CLAUDE.md 的 Learn 工作流中应增加**导航验证步骤**：

```
完成并行写作后，执行导航一致性检查：
1. 读取所有文件的 frontmatter，提取 series.part 编号
2. 按 part 排序，验证文件名与标题的对应关系
3. 检查每章的"下一步学习"链接是否指向实际的下一章文件
4. 检查概览页的路线图和学习路线表是否与实际一致
5. 如发现不一致，立即修复
```
